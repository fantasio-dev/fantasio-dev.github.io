# CSV 1행 → 토픽 페이지(1교시형) 반영 프롬프트 (범용)

## 목적

주어진 CSV 파일의 **특정 1행(토픽 1개)**을 근거로, Jekyll 문서(`docs/{domain}/**`)의 해당 토픽 마크다운 페이지를 **1교시형(요약+표 중심) 템플릿**으로 작성/갱신한다.  
이 프롬프트는 **AI/DS/SEC/DB/BIZ/CAOS/NW/SW 등 모든 도메인**에 재사용한다.

---

## 입력(사용자가 채워 넣을 값)

- **CSV_PATH**: `{/Users/.../기술사_기본필수노트_{DOMAIN} - {DOMAIN}.csv}`
- **DOMAIN_CODE**: `ai | ds | sec | db | biz | caos | nw | sw`
- **ROW_SELECTOR**: 아래 중 하나
  - `NO=13` 처럼 번호로 지정
  - `중토픽="Pooling Layer"` 처럼 중토픽 문자열로 지정
- **TARGET_MD_PATH**: `docs/{domain}/.../xxx.md` (없으면 생성)

---

## 핵심 원칙

- **CSV 내용 우선**: 정의/구성요소/절차/알고리즘은 CSV의 `기술요소`, `매커니즘`을 최대한 그대로 구조화한다.
- **참고자료 금지**: `## 참고자료` 섹션을 만들지 않는다. (기존에 있으면 제거 또는 백업에만 유지)
- **데일리 덱 호환(필수)**: 다음 헤딩/구조를 반드시 포함한다.
  - `### 📌 핵심 암기 (Quick Reference)`  *(헤딩 텍스트에 “암기” 포함)*
  - `### 📝 개념 정의` *(또는 “개념 정의/정의/개념”이 포함된 헤딩)*
  - `### 구성요소` *(또는 “구성요소/기술요소” 포함)*
  - `### 구성요소` 바로 아래에 **표(table) 최소 1개**
- **기존 내용 백업(안전장치)**:
  - 기존 본문이 있으면 맨 아래에 `<details>`로 `🗂️ 기존 내용 (백업)` 섹션으로 감싼다.
  - 단, 이미 `<!-- CSV_APPLIED:` 마커가 있으면 **재백업하지 않고** 교체한다.

---

## 출력 템플릿 (1교시형)

아래 구조를 그대로 사용한다(필수 섹션은 유지):

```markdown
<!-- CSV_APPLIED: {CSV_PATH} | {ROW_SELECTOR} | {TARGET_MD_PATH} -->

# {페이지 타이틀}
{: .fs-8 }

{라벨(있으면 유지)}
```

## CSV → 마크다운 변환 규칙

- 섹션 헤더:
  - CSV 내 `[구성요소]`, `[알고리즘]`, `[절차]`, `[특징]`, `[평가]` 등은 `####` 소제목으로 변환
- 표 변환:
  - `A: 설명` / `A = 설명` / `A > 설명` 형태는 (항목/설명) 표로 변환
  - 항목이 4개 이상이면 우선 표로 만들고, 나머지는 상세 설명에 남긴다.
- 암기법:
  - `암기법` 필드는 공백/줄바꿈 기준으로 토큰화해서 Quick Reference에 백틱(``)으로 나열
  - 가능하면 표에 `암기` 컬럼을 만들어 글자-항목 매핑(가능할 때만)

## 작업 절차(요청 문장 예시)

아래 문장을 그대로 복사해서 사용:

```
@{CSV_PATH}
이 CSV에서 {ROW_SELECTOR} 행을 사용해서
{TARGET_MD_PATH} 페이지를 1교시형 템플릿으로 작성/갱신해줘.

규칙:
- CSV 우선, 참고자료 섹션 금지
- 데일리 덱 호환(암기/개념정의/구성요소 표 필수)
- 기존 본문은 details로 백업(이미 CSV_APPLIED가 있으면 재백업 금지)
- 작업 후 git commit
```
