### 💡 한 줄 정의

> **디지털서비스 전문계약제도**: 공공기관이 클라우드·디지털서비스를 신속하게 도입할 수 있도록 특화된 조달계약 제도

### 🎭 쉬운 비유

| 기존 조달 | 디지털서비스 전문계약제도 |
|:---------|:----------------------|
| 음식점에서 일일이 주문하고 조리 대기 | 편의점에서 바로 픽업하는 도시락 |
| 경쟁입찰 → 심사 → 계약 (40~80일) | 사전심사 완료 제품 → 즉시 계약 (1~2주) |

### ❓ 왜 필요한가? (3줄 요약)

1. **신기술 도입 지연**: 기존 용역계약 절차로는 AI, 클라우드 등 신기술 도입이 어려움
2. **4차 산업혁명 대응**: 선진국은 Cloud First Policy 등으로 클라우드 우선 도입 정책 시행
3. **행정 효율화**: 표준계약서와 디지털서비스몰을 통한 간편한 조달 필요

> - (1) `개념배경` (개념 및 추진배경)
> - (2) `간약사공표` (주요 특징)

> - (4) `소행사혁산` (기대효과)
> - ⭐ **차별점**: 경쟁입찰 없이 수의계약 가능, 디지털서비스몰 통한 원스톱 조달

### 1. 디지털서비스 전문계약제도의 개념 및 추진배경

{: .highlight }
> **개념 및 추진배경**: 클라우드 전환, 4차산업혁명, Cloud First Policy
> - (정의) `전자정부법 제43조의2`
> - (등장배경) `데분디개`
> - (추진배경) `신공계-선`

#### 가. 개념

> **디지털서비스 전문계약제도**: 디지털서비스산업의 변화에 따라 공공기관의 클라우드 컴퓨팅 서비스 도입 활성화를 위해 디지털서비스를 신속하게 도입할 수 있도록 특화된 전문계약제도

| 구분 | 정의 |
|:-----|:-----|
| **디지털서비스** | 클라우드컴퓨팅과 융합기술을 지원하는 서비스 |
| **전문계약제도** | 수의계약·카탈로그 계약 기반 신속 조달 방식 |

#### 나. 추진배경

```
🌐 과거: 공공부문 클라우드 도입 시 경쟁입찰 절차 필수 (40~80일 소요)

💡 문제: "AI, 클라우드 등 신기술을 빠르게 도입하려면?"

✅ 해결: 디지털서비스 전문계약제도 도입!
   → 사전심사 완료 서비스 등록
   → 수의계약으로 신속 도입 (1~2주)
   → Cloud First Policy 실현
```

### 2. 디지털서비스 전문계약제도의 주요 특징

{: .highlight }
> **주요 특징**: 간편조달, 약정운영, 사전심사, 공급자확대, 표준계약서
> - (등록측면) `서종적공의등선서`
> - (유통측면) `서서서`

#### 가. 제도 운영 측면 주요 특징 `간약사공표`

| 구분 | 특징 | 설명 |
|:-----|:-----|:-----|
| **간**편조달 | 경쟁입찰 없이 수의계약 가능 | 조달 절차 간소화 |
| **약**정·운영 | 디지털서비스몰 운영 | 등록부터 계약까지 일괄 처리 |
| **사**전심사제도 도입 | 등록 전 기술·보안성 등 객관적 기준 기반 | 사전 검토 수행 |
| **공**급자 확대 | 중소·스타트업 기업 | 공공시장 진입 기회 확대 |
| **표**준계약서 제공 | 법적 안정성과 계약 일관성 확보 | 분쟁 예방에 기여 |

#### 나. 등록 측면 주요 특징 `서종적공의등선서`

| 구분 | 주요 특징 | 설명 |
|:-----|:---------|:-----|
| **심**사·선정 | **서**면검토 | 전문 검토를 통한 심사 실시 |
| | **종**합심사 | 현장심사를 실시 및 결과 판단 |
| | **적**정성 심사 | 디지털서비스 해당여부 판단 |
| | **공**공역량 심사 | 제공기업의 경영상태, 부정담합자 제재 여부 |
| **등**록·공개 | **의**결 결과·판단 | 서비스 특장점, 수행실적 표기 |
| | **등**록·이용 시스템 등록 | 가격, 인증정보 명시 검토 |
| | **선**정통보서 발급 | 서비스 사양, 운영안정성 확인 |
| | **서**비스 기본정보 제공 | 확장성, 이원제 검증, 보안 |

#### 다. 유통 측면 주요 특징 `서서서`

| 구분 | 주요 특징 | 설명 |
|:-----|:---------|:-----|
| **유**통 | **서**비스 등록 | 디지털 서비스 심사신청 관리 |
| | **서**비스 관리 | 변경사항 관리, 각종 통계 관리 |
| | **서**비스 계약지원 | 정보 및 기능 연계 활용 |

#### 라. 계약 방식 종류

| 종류 | 설명 |
|:-----|:-----|
| **수의 계약** | 디지털서비스 심사위원회 심사를 통해 선정된 디지털 서비스는 전면 수의 계약 가능하도록 개정 (국가를 당사자로 하는 계약에 관한 법률 시행령 제26조, 30조에 의거) |
| **카탈로그 계약** | 수요기관에서 원하는 대로 계약조건을 유연하게 변경할 수 있도록 카탈로그 계약 방식 도입 (조달사업에 관한 법률 시행령 제16조에 의거) |

### 3. 디지털서비스의 종류

{: .highlight }
> **디지털서비스 종류**: 클라우드컴퓨팅서비스, 지원서비스, 융합서비스
> - (대상) `클지융`
> - (클라우드) `I-S-P`
> - (지원서비스) `컨운마`
> - (융합서비스) `빅IoTAI`

#### 가. 디지털서비스 대상 종류

| 종류 | 설명 |
|:-----|:-----|
| **클**라우드컴퓨팅서비스 | 클라우드컴퓨팅을 활용하여 상용으로 타인에게 정보통신자원을 제공하는 서비스 |
| **클**라우드컴퓨팅서비스를 **지**원하는 서비스 | 클라우드컴퓨팅서비스를 지원하는 서비스 (다만, 클라우드컴퓨팅서비스에 해당되는 서비스는 클라우드컴퓨팅서비스로 분류하여 검토) |
| 다른 기술·서비스와 클라우드 컴퓨팅기술을 **융**합한 서비스 | 클라우드컴퓨팅기술·필수역량과 다른 기술·서비스가 융합된 서비스 (다만, 클라우드컴퓨팅서비스에 해당되는 서비스는 클라우드컴퓨팅서비스로 분류하여 검토) |

#### 나. 클라우드컴퓨팅서비스 종류 `I-S-P`

| 구분 | 상세 서비스 | 설명 |
|:-----|:----------|:-----|
| **IaaS** | 서버 서비스 | 특정 서버 환경 구성 적용, 무료/상용 서버 인프라 구성 |
| | 스토리지 서비스 | 블록/스토리지 데이터 운영 환경 |
| **SaaS** | 응용프로그램 서비스 | 앱 실행 서비스, 스케일업 구성 |
| | 컨테이너 서비스 | 프로비저닝 구성 컨테이너 |
| **PaaS** | 애플리케이션 개발/운영 | 앱 개발 환경 제공 |
| | 서버스 | 라이브러리 연동 환경 |
| | 미들웨어 서비스 | 플랫폼 서버스 활용 환경 |

#### 다. 지원서비스와 융합서비스 `컨운마-빅IoTAI`

| 구분 | 상세 서비스 | 설명 |
|:-----|:----------|:-----|
| **지원서비스** | **컨**설팅 서비스 | 공공, 행정, 대국민 업무 특화 데일리킹 |
| | **운**영관리 서비스 | 클라우드 최적화, 맞춤 서비스 모니터링 |
| | **마**이그레이션 서비스 | 기존 행정·공공 서비스 이전, 최적화 |
| **융합서비스** | **빅**데이터 융합서비스 | 대규모 정형·비정형 데이터 수집, 저장, 가공 수행 |
| | **IoT** 융합서비스 | 사물 인터넷 기기 운영, 모니터링 시스템 서비스 |
| | **AI** 융합서비스 | 인공지능 모델 개발, 운영, 유지보수 |

### 4. 디지털서비스 전문계약제도 도입을 통한 기대효과

{: .highlight }
> **기대효과**: 소요시간단축, 행정효율, 사회적비용절감, 혁신시장창출, 산업성장
> - (이용자측면) `기빠행계`
> - (제공자측면) `공시용디`
> - (제도측면) `소행사혁산`

#### 가. 이용자 측면 기대효과 `기빠행계`

| 구분 | 기대효과 | 설명 |
|:-----|:---------|:-----|
| **서비스 이용** | **기**간 단축 | 행정절차 간소화, 소요기간 단축 |
| | **빠**른 서비스 이용 | 불필요 소요시간 감소, 이용계약 체결 후 즉시 이용 |
| **행정효율** | **행**정처리 효율화 | 여러 개 디지털 서비스 묶음 처리 가능 |
| | **계**약 처리 간편 | 하나의 요구사항 정의서로 처리 가능 |

#### 나. 제공자 측면 기대효과 `공시용디`

| 구분 | 기대효과 | 설명 |
|:-----|:---------|:-----|
| **중소기업** | **공**공시장 진입 가속 | 신기술 서비스의 공공시장 판매 가능 |
| **지원** | **시**장 경쟁력 강화 | 완성된 서비스와 제공역량만으로 경쟁 돌입 |
| **제안/응찰** | **용**역계약 모니터링 | 반복제안 없이 1회성 등록으로 수요대응 가능 |
| **간편화** | **디**지털서비스몰 제안응찰 | 수요기관의 처리 효율화 효과 제공 |

#### 다. 제도·산업 측면 기대효과 `소행사혁산`

| 구분 | 특징 | 설명 |
|:-----|:-----|:-----|
| **제도·운영** | **소**요시간 단축 | 수의계약 기반의 간소화된 절차로 계약 및 서비스 도입까지의 시간 절감 |
| | **행**정효율 제고 | 표준화된 계약서와 통합 시스템으로 업무 처리 효율 향상 |
| **산업·시장** | **사**회적비용 절감 | 절차 단축, 오류 감소 등으로 인한 행정법적 비용 감소 |
| | **혁**신적 유효시장 창출 | 공공 수요 기반의 디지털서비스 시장 확대 및 기술기업 진입 장벽 완화 |
| | **산**업성장 견인 | 민간 디지털서비스의 공공 진출 확대를 통해 국내 산업 성장 촉진 |
