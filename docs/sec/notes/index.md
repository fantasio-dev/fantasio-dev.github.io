### 💬 유술사님 피드백

#### 보안 실무 관점
- 보안은 **성실함**이 중요하다.
- 보안은 하찮은 일들이 많다.
- 보안장비는 다 다른 업체꺼를 사용한다.
- **기초적인 부분은 실무적인 느낌이 무조건 나야 한다.**

#### 개별 문제 피드백

| # | 토픽 | 피드백 |
|:--|:-----|:-------|
| 1 | **IGP, EGP** | 알고리즘, 세부 프로토콜을 자세히 쓰면 좋음. **BGP는 공부해 놓을 것**. 라우터 관련해서 많이 나왔었음. [MS BGP 문서](https://learn.microsoft.com/ko-kr/azure/vpn-gateway/vpn-gateway-bgp-overview) |
| 2 | **아티팩트** | 행위를 입증하기 위해서 ~, **디지털 포렌식을 물어본거니까 정의에 디지털 포렌식 어쩌고를 써야 함**. 우선순위 |
| 3 | **암호문 공격** | 암호문 공격이면 **공격 매커니즘**을 적어야함. 보안 토픽은 공격 매커니즘을 적어야 함. 어떤 원리로. 무차별 대입. **커크호프 원칙, 키 관리, 키 강조** → 이런 부분 준비할 것 |
| 4 | **AI 거버넌스** | AI 정책을 기반으로 조직, 프로세스 등이 나온다. AI 윤리도 중요 |
| 5 | **인공지능 시스템의 취약점** | 대응방안은 3단락에 쓰고 **취약점을 위주로 쓸 것** |
| 6 | **클라우드 네이티브 보안** | 보안 솔루션을 더 적는게 좋을 듯 |
| 7 | **공공 마이데이터 활용** | **공공**이라는 도메인에 집중. 상위 토픽이 데이터 주권. 마이데이터 하려면 정책, 거버넌스가 중요하니 언급을 해줄 것 |
| 8 | **OPC UA** | ISO 라고말고 **IEC** 라고 시작해라. 팩토리 쪽이어서 |
| 9 | **보안에서의 샌드박스, 화이트박스** | 허니팟? |
| 10 | **개인정보, 데이터 안심구역** | 법적근거, 운영주체가 다름. 뭔가 **제도스럽게 써야함**. 소공스러운 토픽이어서 절차도 써야함 |
| 11 | **CC 인증** | 가 단락에 전체 구성, 나 단락에 세부 구성. 정보보호제품 평가/인증제도 |
| 12 | **타원곡선 암호화** | (추가 공부 필요) |
| 13 | **OWASP TOP 10** | 가, 나 나누지 말고 그냥 10개 쓰기. **LLM 그림을 크게 그려놓고 전체적인 관점에서 보안을 볼 것**. 그림까지 챙길 것. [참고1](https://blog.aifrica.co.kr/36781), [참고2](https://blog.stackademic.com/owasp-top-10-for-llm-security-2144e6a9d0db) |

## 📌 쿠팡 해킹 사고 분석

### ISMS-P 관점
- ISMS-P 시점에 대해서 체크, 그리고 전체에 대해서 체크하는 것이 아님 (맹점)
- **인력 관리 문제 (관물기)**

### 주요 보안 취약점

| 취약점 유형 | 세부 내용 |
|:------------|:----------|
| **인증키(서명키) 관리 부실** | 퇴사자(중국인 개발자)가 퇴직 후에도 기존 서명키로 가짜 액세스 토큰 생성 가능 |
| **퇴사자 계정 회수 누락** | 내부 시스템 접근권한 폐기·변경 절차 미이행 |
| **장기 유효 인증키 운영** | 키 유효기간 길고 회수되지 않아 6월~11월 5개월간 비정상 접속 지속 |
| **이상 행위 탐지 부재** | 로그 모니터링·비정상 접근 감지 시스템 미작동, 소량씩 장기 데이터 탈취 |
| **접근권한 통제 미흡** | 인증 업무 담당자 권한 과다 부여 및 ISMS-P 관리 조항 미준수 |
