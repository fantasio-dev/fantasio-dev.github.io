### 🔑 핵심 키워드 3개
- **관계(그래프)**: 친구 관계/연결처럼 “누가 누구와 연결됐는지” 구조
- **메시지 패싱(Message Passing)**: 이웃 노드 정보를 모아 내 정보를 업데이트
- **Readout(요약)**: 노드들을 합쳐 그래프 전체의 대표값을 만드는 단계

### 📖 등장배경
- **과거**: CNN/RNN은 격자(이미지)·순서(문장)에는 강하지만, **소셜 네트워크/지식 그래프/분자 구조**처럼 “관계가 핵심인 데이터”는 표현이 어려움
- **문제**: “연결 구조(이웃/연결 강도)를 그대로 살려서 학습하려면?”
- **해결**: **그래프를 그대로 입력**으로 받아, 레이어마다 **이웃 정보를 취합(aggregate)**해서 **노드 표현을 갱신(update)**하고, 필요하면 **그래프 단위로 요약(readout)**하는 GNN

### 📝 개념 정의
> **GNN**: 그래프(노드·엣지) 구조 데이터를 학습해서 **노드/엣지/그래프**의 특성을 예측하는 신경망

### 🧩 기술요소/구성요소 (2그룹 × 4개)

| 그룹 | 세부 항목(4개) |
|:--|:--|
| A (입력 표현) | 인접행렬(Adjacency), 노드특징행렬(Node Feature), 엣지특징(Edge Feature), 라플라시안/스펙트럼(선택) |
| B (학습 흐름) | Message 생성, Aggregate(취합), Update/Combine(업데이트), Readout(생성/요약) |

### ⭐ 차별점 키워드
- **Permutation Invariance(순서 불변성)**: 노드의 “나열 순서”가 바뀌어도 같은 결과를 내야 함

### 🧾 6하원칙 요약
- **누가(Who)**: 각 **노드**가
- **언제(When)**: 각 **레이어**마다
- **어디서(Where)**: 자신의 **이웃 노드**로부터
- **무엇을(What)**: 정보를 받아(메시지)
- **왜(Why)**: 관계를 반영한 표현(embedding)을 만들기 위해
- **어떻게(How)**: `Message → Aggregate → Update → (Readout)`로
- **결과(Result)**: 노드/엣지/그래프 예측 성능 향상

## 🎯 기술사 수준 설명
{: #-기술사-수준-설명}

### 📌 핵심 암기 (Quick Reference)

{: .highlight }
> **GNN**: 그래프관계학습, 메시지패싱, Readout → 그래프 구조 데이터용 신경망
> - (절차) `변취업생` (변환→취합→업데이트→생성/요약)
> - (구성요소) `인노엣라` (인접행렬→노드특징→엣지특징→라플라시안(선택))
> - (모델) `GCN-SAGE-GAT-GIN`

### 🧠 상세 설명

### 1) 개념/정의

| 구분 | 설명 |
|:--|:--|
| **정의** | GNN(Graph Neural Network)은 그래프(노드·엣지)로 표현되는 **관계형/비정형(Non-Euclidean)** 데이터를 학습해 **노드/엣지/그래프** 단위의 예측을 수행하는 모델 |
| **핵심 아이디어** | 이웃 노드로부터 정보를 받아(메시지) 내 표현을 갱신하는 **Message Passing** |
| **핵심 수식(개념)** | \(\mathbf{h}_v^{(k)}=\text{UPDATE}(\mathbf{h}_v^{(k-1)},\text{AGG}(\{\mathbf{h}_u^{(k-1)}|u\in\mathcal{N}(v)\}))\) |

---

### 2) 예측 Tasks (레벨)

| 대상 | 설명 | 예시 |
|:--|:--|:--|
| **노드(Node) 레벨** | 각 노드에 대해 분류/회귀 | 사용자/계정 분류, 문서 분류 |
| **엣지(Edge) 레벨** | 연결 여부/관계 강도 예측 | 링크 예측, 추천(사용자-아이템) |
| **그래프(Graph) 레벨** | 전체 그래프 단위 분류/회귀 | 분자 독성/물성 예측, 그래프 분류 |

---

### 3) 구성요소/기술요소 (2그룹 × 4개) `인노엣라` / `변취업생`

#### 그룹 1: 그래프 입력 표현 `인노엣라`

| 항목 | 설명 | 예시 |
|:--|:--|:--|
| **인접행렬(Adjacency Matrix)** | 노드 간 연결 정보를 표현 | \(A_{uv}=1\)이면 연결 |
| **노드특징행렬(Node Feature Matrix)** | 노드의 속성(피처) | 사용자 프로필 벡터 |
| **엣지특징(Edge Feature)** | 엣지의 타입/가중치/속성 | 관계 타입, 거리, 시간 |
| **라플라시안/스펙트럼(선택)** | 그래프 구조를 스펙트럼 관점에서 표현 | \(L=D-A\) |

#### 그룹 2: 메시지 패싱 파이프라인 `변취업생`

| 항목 | 설명 | 예시 |
|:--|:--|:--|
| **Message** | 이웃에게 전달할 정보 생성 | 선형변환/MLP |
| **Aggregate(취합)** | 이웃 메시지 집계 | Sum/Mean/Max/Attention |
| **Update/Combine** | 취합 결과로 노드 표현 갱신 | GRU/MLP/Residual |
| **Readout(생성/요약)** | 노드→그래프 대표값으로 요약 | Global Pooling |

---

### 4) 특징

| 특징 | 설명 |
|:--|:--|
| **관계 반영** | 데이터 간 관계(엣지)를 직접 반영 |
| **Message Passing** | k-layer면 **k-hop** 이웃 정보까지 반영 가능 |
| **순서 불변성** | 노드 순서가 바뀌어도 동일 결과(집계 함수 설계) |
| **범용성** | 소셜/추천/교통/분자/지식그래프 등 적용 가능 |

---

### 5) 대표 모델 `GCN-SAGE-GAT-GIN`

| 모델 | 설명 |
|:--|:--|
| **GCN** | 그래프 합성곱(스펙트럼/공간) 기반의 대표 모델 |
| **GraphSAGE** | 이웃 샘플링+집계로 대규모 그래프 확장성 강화 |
| **GAT** | Attention으로 이웃 중요도를 다르게 반영 |
| **GIN** | 그래프 구조 구분(표현력) 강화에 초점 |

---

### 6) 활용 사례

- **추천 시스템**: 사용자-아이템 그래프 기반 추천
- **소셜 네트워크 분석**: 커뮤니티 탐지/친구 추천
- **분자/바이오**: 약물-단백질 상호작용, 분자 물성 예측
- **교통/네트워크**: 도로/망 최적화
- **지식 그래프**: 엔티티 관계 학습, QA
