### 🔑 핵심 키워드 3개
- **응집도(Cohesion)**: 같은 군집 안에서 얼마나 “가깝게” 뭉쳤는가
- **분리도(Separation)**: 다른 군집과 얼마나 “멀리” 떨어졌는가
- **\(S(i)\)**: 한 점이 “자기 군집에 잘 속했는지” 점수(-1~1)

### 📖 등장배경
- **과거**: 클러스터링은 정답 레이블이 없어서 “잘 됐는지” 평가가 어려움
- **문제**: 군집 개수 \(K\) 선택이나 알고리즘 비교를 **정량 지표**로 하고 싶음
- **해결**: 군집 내 거리(응집)와 최근접 군집 거리(분리)를 함께 보는 **실루엣 계수**

### 📝 개념 정의
> **실루엣 계수**: 각 데이터가 자기 군집에 얼마나 잘 군집화됐는지(응집도)와 다른 군집과 얼마나 분리됐는지(분리도)를 함께 평가하는 지표

### 🧩 기술요소/구성요소 (2그룹 × 4개)

| 그룹 | 세부 항목(4개) |
|:--|:--|
| A (계산 요소) | 군집 내 평균거리 \(a(i)\), 최근접 군집 평균거리 \(b(i)\), 점수 \(S(i)\), 평균 \(\bar{S}\) |
| B (해석/활용) | 값 범위(-1~1), Clear-cut/Weak cluster, 최적 \(K\) 선택, 알고리즘 비교/이상치 탐지 |

### ⭐ 차별점 키워드
- **라벨 없이 품질 평가**: 비지도 학습에서도 군집 품질을 빠르게 정량 평가 가능

### 🧾 6하원칙 요약
- **누가(Who)**: 각 데이터 포인트 \(i\)가
- **언제(When)**: 클러스터링 결과가 나온 뒤
- **어디서(Where)**: 자기 군집과 가장 가까운 다른 군집을 기준으로
- **무엇을(What)**: 응집도/분리도를
- **왜(Why)**: 군집 품질과 최적 \(K\)를 판단하기 위해
- **어떻게(How)**: `a(i), b(i) → (b-a)/max(a,b)`로 계산

## 🎯 기술사 수준 설명
{: #-기술사-수준-설명}

### 📌 핵심 암기 (Quick Reference)

{: .highlight }
> **실루엣 계수**: 응집도, 분리도, -1~1 → 군집 품질 평가 지표
> - (정의) `응분` (응집도+분리도)
> - (공식) `바아맥` (\(S(i)=(b-a)/max(a,b)\))
> - (활용) `최K비이` (최적K, 비교, 이상치)

### 🧠 상세 설명

### 1) 정의

| 구분 | 내용 |
|:--|:--|
| **정의** | 군집화 결과 평가를 위해 **군집 내 응집도**와 **군집 간 분리도**를 비교하는 거리 기반 지표 |
| **범위** | \(-1 \le S(i) \le 1\) (1에 가까울수록 군집화가 잘 됨) |

---

### 2) 수식(개별 포인트 \(i\))

\[
S(i)=\frac{b(i)-a(i)}{\max(a(i),b(i))}
\]

| 기호 | 의미 |
|:--|:--|
| **\(a(i)\)** | 포인트 \(i\)와 **동일 군집 내 다른 포인트**들과의 평균 거리(응집도) |
| **\(b(i)\)** | 포인트 \(i\)와 **가장 가까운 다른 군집** 포인트들과의 평균 거리(분리도) |

---

### 3) 계산 프로세스

| 프로세스 | 핵심요소 | 설명 |
|:--|:--|:--|
| 개별 포인트간 거리 측정 | 유클리디안 거리(예) | 같은 군집/다른 군집 거리 계산 |
| 군집 내 평균거리 \(a(i)\) | 내부 군집 응집도 | \(i\)와 동일 군집 내 포인트 평균 거리 |
| 최인접 군집 평균거리 \(b(i)\) | 외부 군집 분리도 | \(i\)와 가장 가까운 다른 군집 평균 거리 |
| 실루엣 계수 \(S(i)\) 계산 | \(-1\sim 1\) | 수식으로 \(S(i)\) 산출 |
| 평균 실루엣 계수 \(\bar{S}\) | 군집 품질 평가 | 전체 포인트 \(S(i)\) 평균 |
| 최적 \(K\) 도출 | 최적 \(K\) | 여러 \(K\) 비교 후 \(\bar{S}\)가 큰 \(K\) 선택 |

---

### 4) 해석

| 구분 | 값 | 해석 |
|:--|:--:|:--|
| 개별 \(S(i)\) | \(\approx 1\) | 최적(해당 군집에 잘 속함) |
|  | \(\approx 0\) | 경계(군집 간 거리가 비슷) |
|  | \(\approx -1\) | 잘못됨(이웃 군집이 더 가까움) |
| 평균 \(\bar{S}\) | 높음 | Clear-cut cluster(군집이 명확) |
|  | 낮음 | Weak cluster(군집이 약함) |

---

### 5) 장단점

| 구분 | 설명 |
|:--|:--|
| **장점** | 알고리즘 영향 적음(결과로 평가), 적정 \(K\) 선택 기준 제공, 결과 시각화/해석 용이 |
| **단점** | 모든 포인트 거리 계산 필요로 시간 증가(\(O(n^2)\) 가능), 평균만으로 판단 어려움(군집별 평균도 함께), 밀도 기반(DBSCAN 등)에는 부적합할 수 있음 |

---

### 6) 활용

- **최적 클러스터 수 결정**: K-means에서 \(K\) 선택
- **알고리즘 비교**: K-means vs GMM 등 정량 비교
- **이상치 후보 탐지**: \(S(i)\)가 -1에 가까운 포인트
