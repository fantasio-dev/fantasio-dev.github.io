### 🔑 핵심 키워드 3개

| 키워드 | 설명 |
|:------|:-----|
| **이진 트리** | 각 노드가 최대 2개의 자식 노드를 가지는 트리 |
| **정렬된 배치** | 작은 값은 왼쪽, 큰 값은 오른쪽에 배치 |
| **O(log n)** | 균형 상태에서 검색/삽입/삭제 시간복잡도 |

### 📖 등장배경

| 구분 | 내용 |
|:-----|:-----|
| **과거** | 배열에서 데이터를 찾으려면 처음부터 끝까지 다 확인해야 함 |
| **문제** | 데이터가 많아지면 검색 시간이 너무 오래 걸려요! |

### 📝 개념 정의

| 개념 | 정의 | 쉬운 비유 |
|:-----|:-----|:---------|
| **이진 탐색 트리** | 왼쪽 자식 < 부모 < 오른쪽 자식 규칙의 트리 | 도서관 책 정리 - 작은 번호는 왼쪽 선반 |
| **균형 트리** | 좌우 높이 차이가 적은 트리 | 양팔저울 - 양쪽 무게가 비슷해야 안정적 |

### 🏗️ 기술요소 (2그룹 × 4개)

#### 그룹 1: 구성요소 `루부자리`

| 요소 | 설명 |
|:-----|:-----|
| **루**트 노드 | 트리의 시작점 (최상단) |
| **부**모 노드 | 자식을 가진 노드 |
| **자**식 노드 | 부모에 연결된 하위 노드 |
| **리**프 노드 | 자식이 없는 말단 노드 |

#### 그룹 2: 연산 `삽삭탐순`

| 요소 | 설명 |
|:-----|:-----|
| **삽**입 | 크기 비교 후 적절한 위치에 노드 추가 |
| **삭**제 | 노드 제거 후 트리 구조 재정렬 |
| **탐**색 | 루트부터 크기 비교하며 값 찾기 |
| **순**회 | 중위순회 시 오름차순 정렬 출력 |

### ⭐ 차별점 키워드

**"중위 순회(In-order Traversal)"** - BST를 중위 순회하면 자동으로 오름차순 정렬된 데이터를 얻음

### 🧾 6하원칙 요약

| 항목 | 내용 |
|:-----|:-----|
| **누가** | 데이터베이스, 파일 시스템 등 |
| **언제** | 빠른 검색/삽입/삭제가 필요할 때 |
| **어디서** | 검색 엔진, 정렬 알고리즘, 인덱스 구조 |
| **무엇을** | 정렬된 데이터를 트리 구조로 관리 |
| **어떻게** | 작은 값→왼쪽, 큰 값→오른쪽 규칙 적용 |
| **왜** | O(log n) 시간복잡도로 효율적 연산 |

### 📌 핵심 암기 (Quick Reference)

{: .highlight }
> **이진 탐색 트리(BST)**: 효율적 탐색 자료구조 필요 → 정렬배치, O(log n), 균형유지 → 효율적 검색 트리 자료구조
> - (구성요소) `루부자리왼오` - 루트, 부모, 자식, 리프, 왼쪽서브트리, 오른쪽서브트리
> - (조건) `유전왼오` - 유일값, 전순서, 왼쪽<루트, 오른쪽>루트
> - (연산) `삽삭탐순` - 삽입, 삭제, 탐색, 순회
> - (균형트리) `AVL-RB` - AVL트리, Red-Black트리
> - (비교) `균동검삽시공` - 균형, 동작, 검색시간, 삽입삭제시간, 시간복잡도, 공간복잡도

### 🏗️ 구성요소 (핵심 암기법 상세) `루부자리왼오` / `유전왼오`

#### 그룹 1: BST 구성요소 `루부자리왼오`

| 암기 | 구분 | 구성요소 | 설명 | 예시 |
|:-----|:-----|:---------|:-----|:-----|
| **루** | 노드 | 루트(root) 노드 | 트리 최상단 노드, 탐색의 시작점 | 트리의 첫 번째 노드 |
| **부** | 노드 | 부모(parent) 노드 | 분기를 통해 계층적 연결을 이루는 상위 노드 | Edge로 연결 |
| **자** | 노드 | 자식(child) 노드 | 부모에 연결된 하위 노드 | 최대 2개 |
| **리** | 노드 | 리프(leaf) 노드 | 자식 노드가 없는 말단 노드 | NULL 자식 |
| **왼** | 서브트리 | 왼쪽 서브트리 | 루트 기준 왼쪽, 루트보다 작은 값들 | Left Subtree |
| **오** | 서브트리 | 오른쪽 서브트리 | 루트 기준 오른쪽, 루트보다 큰 값들 | Right Subtree |

#### 그룹 2: BST 조건 `유전왼오`

| 암기 | 구분 | 조건 | 설명 | 예시 |
|:-----|:-----|:-----|:-----|:-----|
| **유** | 자료 | 유일한 값 | 각 노드의 값은 유일한 식별자로 정수로 존재 | 중복값 없음 |
| **전** | 자료 | 전순서 존재 | 임의의 두 원소를 비교할 수 있는 부분 순서 집합 | 허수/복소수 불가 |
| **왼** | 구조 | 왼쪽 < 루트 | 이진 탐색 트리 값 삽입시 루트 값과 비교, 작으면 왼쪽 | 왼쪽 서브트리 이동 |
| **오** | 구조 | 오른쪽 > 루트 | 이진 탐색 트리 값 삽입시 루트 값과 비교, 크면 오른쪽 | 오른쪽 서브트리 이동 |

## ⭐ 차별점 키워드 (가산점 포인트)

{: .important }
> **중위 순회(In-order Traversal)와 정렬**
> - BST를 중위 순회(왼쪽 → 루트 → 오른쪽)하면 **오름차순 정렬**된 값 출력
> - 재귀적 성격: 서브트리도 BST 구조를 그대로 유지
> - 컴퓨터 디렉토리 구조 등의 계층구조로 AVL 트리, Red-Black 트리 사용

#### 📊 균형 유지형 이진 탐색 트리

##### AVL 트리

```
                Balance Factor = 2-2 = 0
                        ┌───┐
                        │10 │
                        └─┬─┘
              ┌───────────┴───────────┐
              │  BF=0-1=-1            │
          ┌───┴───┐               ┌───┴───┐
          │  7    │               │  13   │
          └───┬───┘               └───────┘
      ┌───────┴───────┐
  ┌───┴───┐       ┌───┴───┐
  │  3    │       │  5    │        ┌───┐
  └───────┘       └───────┘        │20 │
                                   └───┘
```

| 특징 | 설명 |
|:-----|:-----|
| **Balance Factor** | 왼쪽 서브트리 높이 - 오른쪽 서브트리 높이 (절대값 ≤ 1) |
| **시간복잡도** | 검색, 삽입, 삭제 모두 O(log n) |
| **균형 기법** | LL, RR, RL, LR 회전(Rotation) |

##### Red-Black 트리

```
                        ┌─────┐
                        │ 10  │ ← Black
                        │(B)  │
                        └──┬──┘
              ┌────────────┴────────────┐
          ┌───┴───┐                 ┌───┴───┐
          │  7    │ ← Red           │  13   │ ← Red
          │ (R)   │                 │ (R)   │
          └───┬───┘                 └───┬───┘
      ┌───────┴───────┐         ┌───────┴───────┐
  ┌───┴───┐       ┌───┴───┐ ┌───┴───┐       ┌───┴───┐
  │  3    │       │  5    │ │ NIL   │       │ NIL   │
  │ (B)   │       │ (B)   │ │ (B)   │       │ (B)   │
  └───┬───┘       └───┬───┘ └───────┘       └───────┘
      │               │
  ┌───┴───┐       ┌───┴───┐
  │ NIL   │       │ NIL   │    * NIL = Black Leaf
  └───────┘       └───────┘
```

| 특징 | 설명 |
|:-----|:-----|
| **Red 노드** | 부모, 양쪽 자식은 반드시 Black 노드 |
| **Black 노드** | 루트 노드, 모든 리프 노드(NIL)는 Black |
| **Black 높이** | 특정 노드에서 리프까지 모든 경로의 Black 노드 개수 동일 |
| **균형 기법** | Restructuring(회전), Recoloring(색 변경) |

### 🔄 삽입/삭제 연산

#### 삽입 절차 `루비작오빈`

| 암기 | 단계 | 절차 | 설명 |
|:-----|:-----|:-----|:-----|
| **루** | 1 | 루트에서 시작 | 루트 노드부터 탐색 시작 |
| **비** | 2 | 비교 | 새 값과 현재 노드 값 비교 |
| **작** | 3 | 작으면 왼쪽 | 새 값 < 현재 노드 → 왼쪽 서브트리로 이동 |
| **오** | 4 | 오른쪽 | 새 값 > 현재 노드 → 오른쪽 서브트리로 이동 |
| **빈** | 5 | 빈 자리 삽입 | 빈 자리 찾으면 노드 삽입 |

#### 삭제 절차 `단자이후대`

| 암기 | 경우 | 절차 | 설명 |
|:-----|:-----|:-----|:-----|
| **단** | 단말 노드 | 바로 삭제 | Leaf Node는 바로 삭제 |
| **자** | 자식 1개 | 자식을 부모에 연결 | 자식 노드를 삭제된 노드 위치로 승계 |
| **이** | 이개 (자식 2개) | 후속자 찾기 | 오른쪽 서브트리에서 최소값(후속자) 탐색 |
| **후** | 후속자 대체 | 값 대체 | 후속자 값으로 삭제 노드 대체 |
| **대** | 대체 후 삭제 | 후속자 삭제 | 후속자 노드 삭제 |

### 📊 AVL 트리 vs Red-Black 트리 비교 `균동검삽시공`

| 암기 | 비교 항목 | AVL 트리 | Red-Black 트리 |
|:-----|:---------|:---------|:---------------|
| **균** | 균형 | **엄격한 균형** (높이 차이 ≤ 1) | **느슨한 균형** (Black 높이만 동일) |
| **동** | 동작 | LL, RR, RL, LR 회전 | Restructuring, Recoloring |
| **검** | 검색 시간 | **상대적 빠름** (엄격한 균형) | 상대적 느림 |
| **삽** | 삽입/삭제 시간 | 상대적 느림 (잦은 회전) | **상대적 빠름** |
| **시** | 시간복잡도 | O(log n) | O(log n) |
| **공** | 공간복잡도 | 노드당 int Bit (Balance Factor 값) | **노드당 1 Bit** (Red or Black) |
