## 다. 초거대 AI의 도입절차

### 📋 도입절차 `보클학서운성`

| 단계 | 도입절차 | 주요 내용 |
|:-----|:--------|:---------|
| 1 | **보**안등급 (데이터 보안 등급) | 업무 중요도에 따라 C(기밀)/S(민감)/O(공개) 3개 등급으로 분류하여 보안정책 적용 |
| 2 | **클**라우드 구성 방안 | 클라우드 영역 및 규모 선정, 클라우드 도입유형 결정 등 서비스 구성 |
| 3 | **학**습방식 (데이터 학습 방식) | 파운데이션 모델, 파인튜닝 모델, 사후 학습 모델, RAG 기반 모델로 구분 |
| 4 | **서**비스 도입 방식 | 디지털 서비스 구매(클라우드 컴퓨팅/융합서비스) 및 조달 용역발주 방식 추진 |
| 5 | **운**영 (유지보수 및 운영) | 데이터 준비, 모델 구축, 초기 설정, 사전학습, 추가학습, 교육, 배포, 모니터링, 최적화, 거버넌스 체계 마련 |
| 6 | **성**과관리 | AI 과제의 체계적인 성과 관리를 위해 성과지표 설정 및 관리 |

---

### 📊 C/S/O 분류 기준

| 구분 | 분류 기준 | 설명 |
|:-----|:---------|:-----|
| **C**(기밀정보) | 비밀, 안보·국방·외교수사 등 기밀정보 및 국민 생활·안전과 직결된 정보 | 제1호: 법률상 비밀·비공개로 규정, 제2호: 안보·국방·통일·외교 관련 공개 시 국익 저해, 제3호: 국민 생명·신체·재산·보호에 현저한 지장, 제4호: 진행중 재판 및 범죄예방·수사·공소형·교정 관련 정보, 제5호: 감사·감독·시험·입찰계약·기술개발·인사관리 및 의사결정·내부검토 관련 정보 |
| **S**(민간정보) | 비공개 정보로 개인·국가 이익 침해가 가능한 정보 | 제6호: 설명·주민번호 등 개인정보, 공개 시 사생활 침해, 제7호: 법인·단체·개인의 경영상·영업상 비밀로, 공개 시 이익 침해, 제8호: 공개 시 부동산투기, 매점매석으로 특정인에게 이익·불이익, 기타: 로그 및 임시백업 등 |
| **O**(공개정보) | 기밀·민감정보 이외 모든 정보 및 별도의 조치를 적용한 비공개 정보 | 공공데이터법(제2조)에 따른 공공데이터로 기밀·민감·정보 이외 모든 정보, 관련 법령 등에서 규정하는 요건을 조치한 행정·민감 정보, 기관의 경과 등으로 비공개 필요성 소멸 시 공개한 정보 |

### 📋 서비스 레벨 목표(SLO) 주요내용

| 구분 | 주요내용 |
|:-----|:--------|
| **BLEU** | 사람이 직접 번역한 문장과의 정확도 비교를 통한 품질 측정 |
| **응답 시간** | 사용자 질의에 대한 LLM 응답 속도 목표 (예: 문답 90% 이상이 7초 이내) |
| **가용성** | LLM 서비스 가용/접근 가능 시간 목표 지표 (예: 월 가용성 99.99%) |
| **데이터 보안** | 사용자 데이터와 LLM 학습 데이터의 암호화/접근통제 수준 명시 |
| **윤리강령 준수** | 편향 및 위험의 검증 절차, 제한 준수 규정 |
| **모델 업데이트 주기** | 정기 모델 업데이트 주기, 모델 향상 주기 |
| **책임과 보상 체계** | 서비스 문제 발생 시 책임 소재 및 보상 체계 |
| **사용자 수용능력** | 동시 접속 가능한 최대 사용자 수 |
| **기술 지원** | 기술 문의에 대한 지원 채널 제공, 지원 시간 및 초기 응답 시간 목표 |
| **사용 데이터 보고** | 사용량 및 성과 추이 데이터에 대한 정기 보고 |
| **해지 정책** | 계약 해지 시 데이터 처분, 모델 이전 등에 대한 정책 |
| **법적 준수 사항** | LLM 서비스의 법률 및 규제 준수에 대한 보장 |
