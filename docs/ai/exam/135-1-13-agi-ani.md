## ✅ 문제

AGI(Artificial General Intelligence) 측면에서 ANI(Artificial Narrow Intelligence)의 필요성

### 🔑 핵심 키워드 3개
- **ANI → AGI**: ‘특정 분야 잘하는 AI(ANI)’를 모아 ‘범용 AI(AGI)’로 가는 단계
- **데이터/기술 축적**: ANI로 만든 데이터·지식·기술이 AGI의 재료가 됨
- **비용/리스크 절감**: 한 번에 AGI를 만들기보다 단계적으로 검증하며 안전하게 확장

### 📖 등장배경
- **과거**: AI는 먼저 “한 가지 분야를 아주 잘하는” 형태(ANI)로 성공 사례를 만들었음
- **문제**: AGI는 범용 문제 해결이 목표라 비용·시간·리스크가 매우 큼(성능/안전/윤리/법)
- **해결**: ANI를 통해 **학습 데이터·도메인 지식·기반 기술·검증 방법**을 쌓고, 이를 바탕으로 AGI로 **점진적 확장**한다

### 📝 개념 정의
> **ANI(Artificial Narrow Intelligence)**: 특정 목적/도메인에 특화된 작업을 수행하도록 설계된 인공지능  
> **AGI(Artificial General Intelligence)**: 다양한 작업을 인간처럼 일반화하여 해결하는 범용 인공지능

### 🧩 기술요소/구성요소 (2그룹 × 4개)

| 그룹 | 세부 항목(4개) |
|:--|:--|
| A (개발 관점 필요성) | 데이터 확보/학습, 전문지식 축적, 기반기술 마련(NLP/CV/ASR 등), 개발시간 단축(전이학습/재사용) |
| B (비용·신뢰·검증) | 개발비 감소(모듈화), 단계적 R&D 투자, 윤리/신뢰 사전 점검, 도메인별 평가·검증 체계 구축 |

### ⭐ 차별점 키워드
- **점진적 접근(Incremental Path)**: “완전한 AGI를 한 번에”가 아니라 **검증된 ANI를 기반으로 확장**

### 🧾 6하원칙 요약
- **누가(Who)**: 조직/연구자가
- **언제(When)**: AGI 개발/도입을 준비할 때
- **어디서(Where)**: 도메인(특정 분야) 문제 공간에서 ANI로
- **무엇을(What)**: 데이터·기술·검증체계를
- **왜(Why)**: 비용/리스크를 줄이고 성공 확률을 높이기 위해
- **어떻게(How)**: `데전기시`로 쌓고 `비단윤평`으로 검증하며 확장

## 🎯 기술사 수준 설명
{: #-기술사-수준-설명}

### 📌 핵심 암기 (Quick Reference)

{: .highlight }
> **ANI의 필요성(AGI 관점)**: 데이터·기술 축적, 비용 절감, 검증 기반 → AGI로 가는 점진적 경로
> - (개발) `데전기시` (데이터확보/학습, 전문지식, 기반기술, 시간단축)
> - (비용·신뢰) `비단윤평` (비용감소, 단계투자, 윤리/신뢰, 평가/검증체계)
> - (비교) `특학적` (특징, 학습방식, 적응성)

### 1) ANI 개념(AGI/ASI 관점 위치)

| 구분 | 내용 |
|:--|:--|
| **레벨** | **ANI → AGI → ASI** (특화 → 범용 → 초지능) |
| **ANI 정의** | 특정 분야에 특화된 약한 인공지능(사전 설정된 작업 중심) |
| **ANI 한계** | 범용적 사고/새 문제 해결에 제한(도메인 밖 일반화 어려움) |

---

### 2) AGI 관점에서 ANI가 필요한 이유 (2그룹 × 4개) `데전기시` / `비단윤평`

#### 그룹 1: 개발 측면 필요성 `데전기시`

| 항목 | 필요성 | 설명 |
|:--|:--|:--|
| **데이터 확보/학습(데)** | AGI의 기반 데이터 | ANI로 수집된 방대한 정보가 AGI 학습의 기본 재료 |
| **전문 지식 축적(전)** | 도메인 해결 능력 | 분야별 전문지식을 AGI에 재사용해 문제 해결력 강화 |
| **기반 기술 마련(기)** | 핵심 요소기술 성숙 | NLP/CV/ASR 등 AGI에 필요한 요소기술을 ANI에서 검증·성숙 |
| **시간 단축(시)** | 개발 효율 | 전이학습/모듈 재사용으로 AGI 개발 시간 단축 |

#### 그룹 2: 비용/신뢰/검증 필요성 `비단윤평`

| 항목 | 필요성 | 설명 |
|:--|:--|:--|
| **비용 감소(비)** | AGI 개발비 절감 | ANI를 모듈로 활용해 AGI 비용을 낮춤 |
| **단계적 투자(단)** | R&D 리스크 분산 | AGI를 한 번에 만들기보다 단계적 확장으로 투자 효율화 |
| **윤리/신뢰(윤)** | 안전·윤리 기준 사전 점검 | 윤리 가이드라인/안전 설계 원칙을 ANI에서 먼저 검증 |
| **평가/검증 체계(평)** | 도메인별 검증을 AGI로 확장 | 도메인별 성능 평가 방법론을 구축 후 AGI 검증에 활용 |

---

### 3) AGI vs ANI 비교 `특학적`

| 비교 | AGI(범용) | ANI(특화) |
|:--|:--|:--|
| **특징(특)** | 다양한 작업을 범용적으로 처리 | 특정 분야에 특화 |
| **학습 방식(학)** | 다양한 학습 방식 통합/일반화 | 제한된 범위 내 학습 |
| **적응성(적)** | 상황 변화에 자기 적응/학습 | 정해진 작업 내에서만 적용 |
