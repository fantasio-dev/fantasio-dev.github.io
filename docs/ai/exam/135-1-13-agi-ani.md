### 🔑 핵심 키워드 3개

| 키워드 | 설명 |
|:------|:-----|
| **ANI → AGI** | '특정 분야 잘하는 AI(ANI)'를 모아 '범용 AI(AGI)'로 가는 단계 |
| **데이터/기술 축적** | ANI로 만든 데이터·지식·기술이 AGI의 재료가 됨 |
| **비용/리스크 절감** | 한 번에 AGI를 만들기보다 단계적으로 검증하며 안전하게 확장 |

### 📖 등장배경

| 구분 | 내용 |
|:-----|:-----|
| **과거** | AI는 먼저 "한 가지 분야를 아주 잘하는" 형태(ANI)로 성공 사례를 만들었음 |
| **문제** | AGI는 범용 문제 해결이 목표라 비용·시간·리스크가 매우 큼(성능/안전/윤리/법) |

### 📝 개념 정의

| 개념 | 정의 | 쉬운 비유 |
|:-----|:-----|:---------|
| **ANI** | 특정 목적/도메인에 특화된 작업을 수행하도록 설계된 인공지능 | 바둑만 잘 두는 알파고 |
| **AGI** | 다양한 작업을 인간처럼 일반화하여 해결하는 범용 인공지능 | 모든 분야에서 사람처럼 생각하는 AI |

### 🧩 기술요소 (2그룹 × 4개)

#### 그룹 1: 개발 관점 필요성 `데전기시`

| 요소 | 설명 |
|:-----|:-----|
| **데**이터 확보/학습 | ANI로 수집된 데이터가 AGI 학습 재료 |
| **전**문지식 축적 | 분야별 전문지식을 AGI에 재사용 |
| **기**반기술 마련 | NLP/CV/ASR 등 요소기술을 ANI에서 검증 |
| **시**간 단축 | 전이학습/재사용으로 AGI 개발 시간 단축 |

#### 그룹 2: 비용·신뢰·검증 `비단윤평`

| 요소 | 설명 |
|:-----|:-----|
| **비**용 감소 | ANI를 모듈로 활용해 AGI 비용 절감 |
| **단**계적 투자 | R&D 리스크 분산, 투자 효율화 |
| **윤**리/신뢰 | 안전·윤리 기준을 ANI에서 먼저 검증 |
| **평**가/검증 체계 | 도메인별 평가 방법론 구축 후 AGI에 활용 |

### ⭐ 차별점 키워드

**"점진적 접근(Incremental Path)"** - 완전한 AGI를 한 번에 만들지 않고 검증된 ANI를 기반으로 확장

### 🧾 6하원칙 요약

| 항목 | 내용 |
|:-----|:-----|
| **누가(Who)** | 조직/연구자가 |
| **언제(When)** | AGI 개발/도입을 준비할 때 |
| **어디서(Where)** | 도메인(특정 분야) 문제 공간에서 ANI로 |
| **무엇을(What)** | 데이터·기술·검증체계를 |
| **왜(Why)** | 비용/리스크를 줄이고 성공 확률을 높이기 위해 |
| **어떻게(How)** | `데전기시`로 쌓고 `비단윤평`으로 검증하며 확장 |

### 📌 핵심 암기 (Quick Reference)

{: .highlight }
> **ANI의 필요성(AGI 관점)**: AGI 실현 위한 단계적 접근 필요 → 데이터·기술 축적, 비용 절감, 검증 기반 → AGI로 가는 점진적 경로
> - (개발) `데전기시` - 데이터확보/학습, 전문지식, 기반기술, 시간단축
> - (비용·신뢰) `비단윤평` - 비용감소, 단계투자, 윤리/신뢰, 평가/검증체계
> - (비교) `특학적` - 특징, 학습방식, 적응성

| **평** | 평가/검증 체계 | 도메인별 검증을 AGI로 확장 | 도메인별 성능 평가 방법론을 구축 후 AGI 검증에 활용 |

## ⭐ 차별점 키워드 (가산점 포인트)

{: .important }
> **점진적 접근(Incremental Path)**
> - 완전한 AGI를 한 번에 개발하는 것이 아니라 **검증된 ANI를 기반으로 확장**
> - ANI에서 축적된 데이터, 기술, 검증 체계가 AGI 개발의 핵심 자산
> - 비용/리스크 분산과 함께 윤리적 검증도 단계적으로 진행 가능

### 📊 AGI vs ANI 비교 `특학적`

| 암기 | 비교 | AGI(범용) | ANI(특화) |
|:-----|:-----|:---------|:---------|
| **특** | 특징 | 다양한 작업을 범용적으로 처리 | 특정 분야에 특화 |
| **학** | 학습 방식 | 다양한 학습 방식 통합/일반화 | 제한된 범위 내 학습 |
| **적** | 적응성 | 상황 변화에 자기 적응/학습 | 정해진 작업 내에서만 적용 |
