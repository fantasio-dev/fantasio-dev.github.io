### 🔑 핵심 키워드 3개

| 키워드 | 설명 |
|:------|:-----|
| **목표 지향(Goal-driven)** | 목표를 스스로 세우고 달성하려고 함 |
| **계획+실행(Plan & Act)** | 계획을 세우고 실제 도구/API로 실행까지 함 |
| **피드백 학습(Feedback Loop)** | 결과를 보고 다음 행동을 계속 개선 |

### 📖 등장배경

| 구분 | 내용 |
|:-----|:-----|
| **과거** | 챗봇/비서형 LLM은 "답변"은 잘하지만, 업무를 끝까지 "수행"하는 데는 한계 |
| **문제** | 여러 단계를 거쳐 목표를 달성해야 하는 업무는 **계획·도구·상태 추적**이 필요 |

### 📝 개념 정의
> **에이전틱 AI(Agentic AI)**: 목표 설정과 자율 계획 수립을 통해 **인간처럼 스스로 의사결정하고 행동(도구 실행)하는 AI**

### 🧩 기술요소/구성요소 (2그룹 × 4개)

| 그룹 | 세부 항목(4개) |
|:--|:--|
| A (작동 루프) | 인식(Perceive), 추론(Reason), 행동(Act), 학습(Learn) |
| B (핵심 구성) | LLM, 메모리(DB/VectorDB), 도구(Action/API), 데이터 플라이휠(Data Flywheel) |

### ⭐ 차별점 키워드
- **High Agency**: 단순 반응형이 아니라 **자율적·선제적 계획**으로 복잡한 목표를 수행

### 🧾 6하원칙 요약

| 항목 | 내용 |
|:-----|:-----|
| **누가(Who)** | 에이전트(Agent)가 |
| **언제(When)** | 목표가 주어지면/스스로 목표를 정하면 |
| **어디서(Where)** | 외부 시스템(API/DB/SaaS/IoT 등)과 연결된 환경에서 |
| **무엇을(What)** | 작업을 |
| **왜(Why)** | 목표 달성과 업무 자동화를 위해 |
| **어떻게(How)** | `인추행학` 루프로 수행하고 `LLM-메도데`로 구성 |

## 🎯 기술사 수준 설명
{: #-기술사-수준-설명}

### 📌 핵심 암기 (Quick Reference)

{: .highlight }
> **에이전틱 AI**: 목표지향, 계획·도구실행, 피드백학습 → 자율 의사결정·행동 AI
> - (루프) `인추행학` - Perceive→Reason→Act→Learn
> - (구성) `LLM-메도데` - LLM→메모리→도구→데이터플라이휠
> - (특징) `자목상반` - 자율성, 목표지향, 상황인식/적응, 반복학습
> - ⭐ **차별점**: High Agency: 단순 반응형이 아니라 **자율적·선제적 계획**으로 복잡한 목표를 수행

## 🏗️ 기술 영역

<div class="exam-tech-block" markdown="1">

### 🏗️ 구성요소 (핵심 암기법 상세) `인추행학` / `LLM-메도데`

#### 그룹 1: 작동 루프 `인추행학`

| 암기 | 단계 | 기술요소 | 설명 |
|:-----|:-----|:--------|:-----|
| **인** | 인식(Perceive) | 데이터 수집, 특징 추출 | 센서/DB/API 등에서 데이터를 수집하고 유의미한 패턴을 추출 |
| **추** | 추론(Reason) | LLM 기반 추론, RAG | 작업을 이해하고 솔루션을 생성하며, 외부 지식 접근 |
| **행** | 행동(Act) | 목표 설정·자율 계획, API 통합 | 계획에 따라 외부 소프트웨어와 상호작용하며 작업 수행 |
| **학** | 학습(Learn) | 피드백 루프, 데이터 플라이휠 | 실행 결과를 평가·학습해 성능을 지속 개선 |

#### 그룹 2: 시스템 구성 `LLM-메도데`

| 암기 | 구성 | 역할 | 예시 |
|:-----|:-----|:-----|:-----|
| **LLM** | LLM | 목표/계획/추론의 중심 두뇌 | 계획 생성, 정책 준수 |
| **메** | 메모리 | 장기 지식/상태 저장 | Database, Vector DB |
| **도** | 도구(Action) | 외부 시스템 실행 수단 | API, Webhook, 자동화 툴체인 |
| **데** | 데이터 플라이휠 | 운영 데이터로 지속 개선 | 사용자 상호작용 로그→개선 |

## ⭐ 차별점 키워드 (가산점 포인트)

{: .important }
> **High Agency (고수준 자율성)**
> - 단순 반응형이 아니라 **자율적·선제적 계획**으로 복잡한 목표를 수행
> - "답변형 AI"가 아니라 **"수행형(행동형) AI"**
> - Gartner 2025 전략기술 트렌드 선정

```
🌐 과거: 챗봇/비서형 LLM은 "답변"은 잘하지만, 업무를 끝까지 "수행"하는 데는 한계

💡 문제: "여러 단계를 거쳐 목표를 달성해야 하는 업무는 계획·도구·상태 추적이 필요"

✅ 해결: 에이전틱 AI로 자율 수행!
   → LLM에 메모리(지식), 계획, 환경 인식, 도구(Action)을 결합
   → 결과 피드백을 통해 성능을 높이는 자율형 AI
   → Gartner 2025 전략기술 트렌드 선정
```

---

#### 🔄 작동 방식(예시 6단계)

| 단계 | 동작 원리(요지) |
|:-----|:---------------|
| ① 인식 | 센서/API/DB/사용자 상호작용으로 최신 데이터 수집 |
| ② 추론 | LLM/RAG/인지 기술로 맥락 이해 |
| ③ 목표 설정 | 사용자 입력/정책 기반 목표 정의 |
| ④ 의사결정 | 효율·정확·예측결과 등을 고려해 최적 행동 선택 |
| ⑤ 실행 | API/로봇/시스템과 연동해 작업 수행 |
| ⑥ 학습·적응 | 결과 평가·피드백으로 다음 계획/행동 개선 |

---

#### 📊 에이전트 유형(구현 관점)

| 구분 | 유형 | 설명(요지) |
|:-----|:-----|:----------|
| 규칙 기반 | 단순 반사 에이전트 | 규칙 기반 즉각 반응(지각/행동 중심) |
|  | 모델 기반 에이전트 | 내부 모델로 상태 저장 후 의사결정 |
| 목표 기반 | 목표 기반 에이전트 | 목표 달성 계획 수립+실행(지각/추론/행동) |
|  | 유틸리티 기반 에이전트 | 효용 함수로 행동 최적화(판단 강화) |
| 학습 기반 | 학습 에이전트 | 경험/피드백으로 성능 개선 |
| 계층 기반 | 계층적 에이전트 | 복잡한 작업을 하위 작업으로 분해 |

#### 💼 주요 사례

| 분야 | 주요사례 | 설명 |
|:-----|:--------|:-----|

| 보안 강화 | 보안 취약점 분석, 금융 거래 감시 | 위협 탐지 및 대응 자동화, 이상 패턴 탐지 |
| 의료 지원 | 진단/관리 보조, 원격 모니터링 | 업무 자동화로 효율 향상, 이상 징후 알림 |
| 소매/서비스 | 개인화 서비스, 재고·주문 예측 | 고객 만족/운영 효율 극대화 |
