# AI 이해 가이드라인 (MAP)
{: .no_toc }

`AI - MAP.csv`를 기반으로 만든 **AI 큰그림(시험 답안용) 가이드**입니다.  
목표는 “많은 키워드를 **올바른 형식**으로 배치해서” 빠르게 서술하는 것입니다.

---

## 🎯 핵심 암기 (Quick Reference)

{: .highlight }
> **AI 개발 Lifecycle / 머신러닝 파이프라인**: 데이터 → 모델 → 운영(피드백)으로 이어지는 end-to-end 흐름
> - (데이터) `수저처` 수집·저장·전처리
> - (모델) `선학평` 선정·학습·평가
> - (운영) `배모튜` 배포·모니터링·튜닝
>
> **AI 학습 단계**: `인추판제행학` 인식→추론→판단→제어→행동→학습 (+ 피드백 루프)

---

## 🧭 한 장 요약: Lifecycle (데이터 → 모델 → 운영)

```text
데이터(수저처)              모델(선학평)              운영(배모튜)
┌──────────────────┐      ┌──────────────────┐      ┌──────────────────┐
│ [데이터 수집]     │ ---> │ [모델 선정]       │ ---> │ [모델 배포]       │
│       ↓          │      │       ↓          │      │       ↓          │
│ [데이터 저장]     │      │ [모델 학습]       │      │ [모니터링/로그]   │
│       ↓          │      │       ↓          │      │       ↓          │
│ [전처리]          │      │ [모델 평가]       │      │ [튜닝]            │
└──────────────────┘      └──────────────────┘      └──────────────────┘
```

### 용어 메모

- **모델 서빙(Serving)**: 모델을 API/서비스 형태로 제공
- **딜리버리(Delivery)**: 배포/릴리즈 관점의 전달
- **Data Drift**: 입력 데이터 분포 변화
- **GPUaaS**: GPU를 서비스로 제공

---

## 🗃️ 데이터 파이프라인 (Data Pipeline)

```text
데이터 수집 → 데이터 저장 → 데이터 처리 → 데이터 분석 → 데이터 시각화
```

{: .important }
> 인공지능은 항상 **“데이터 품질”**과 연결됨

## 📏 모델 평가: 키워드 체크리스트

| 구분 | 키워드 | 설명 |
|:--|:--|:--|
| 데이터 분리 | Training/Validation/Test | 일반화 성능 확인 |
| 검증 방법 | Hold-out / K-Fold / LOOCV | 반복 평가 |
| 평가 지표 | Accuracy/Precision/Recall/F1/RMSE | 수치화 |
| 과적합 확인 | Overfitting / Underfitting | 일반화 문제 |
| 베이스라인 | Baseline | 개선 효과 비교 |
| 튜닝 | Grid/Random/Bayesian | 파라미터 최적화 |
| 모델 비교 | Model Selection | 최적 모델 선택 |
| 신뢰성 | Robustness/Fairness/Bias | 공정/안정 |
| 시각화 | Confusion Matrix/ROC/Learning Curve | 시각 분석 |
