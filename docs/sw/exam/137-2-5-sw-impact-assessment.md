> **SW 영향평가**: 국가기관의 SW사업이 민간 시장을 침해하는지 사전 검토하는 제도

### 쉬운 비유

| 상황 | 비유 |
|:-----|:-----|
| 정부가 무료 SW 배포 | 대형마트가 동네 상권에 들어오는 것 |
| 영향평가 | 상권 영향 분석 후 진출 조정 |

### 왜 필요한가? (3줄 요약)

1. **민간 시장 보호**: 정부의 무상 SW 배포로 인한 민간 기업 피해 방지
2. **예산 낭비 방지**: 공공-민간 중복 개발 사전 차단
3. **SW 산업 육성**: 민간 SW 기업의 경쟁력 확보 지원

### 1. 소프트웨어사업 영향평가 개념

{: .highlight }
> **개념**: 민간 SW시장 침해 예방, 사전 조정, SW 산업 육성
> - (법적 근거) `진흥법 제43조, 시행령 제35~37조`
> - (목적) `침예육` (침해예방, 예산낭비방지, 육성)
> - (도입배경) `무무직` (무상배포, 무상제공, 직접경쟁)

#### 가. 개념 정의

| 구분 | 내용 |
|:-----|:-----|
| **정의** | 국가기관 등에서 SW사업의 예산편성, 발주, SW 배포 및 서비스 제공을 추진하는 경우 민간 SW 시장 침해 등 SW 산업 생태계에 미치는 영향을 검토하여 사전 조정하는 제도 |
| **법적 근거** | 「소프트웨어 진흥법」 제43조, 시행령 제35조(영향평가 및 재평가의 실시), 제36조(결과 등의 공시 등), 제37조(결과서등의 검토 및 개선조치 협조 등) |

#### 나. 도입배경

```
🌐 과거: 국가기관의 SW 무상 배포 및 민간 직접 경쟁 심화

💡 문제: "민간 SW 시장을 위축시켜 산업 육성을 저해하지 않는가?"

  - 정보시스템의 기관 간 무상배포
  - 패키지 소프트웨어를 국민에게 무상배포
  - 특정분야에서 민간기업이 제공하는 소프트웨어와 직접 경쟁

✅ 해결: SW 영향평가 제도 도입!
   → 예산편성 단계부터 민간 침해 가능성 검토
   → 공공-민간 중복 개발 방지
```

### 2. 소프트웨어사업 영향평가 대상 기관

{: .highlight }
> **대상 기관**: 진흥법 시행령 제21조(국가기관등의 범위) 각 호에 규정된 기관
> - (공공) `국지공` (국가기관, 지방자치단체, 공공기관)
> - (출연) `정과정출지` (정부출연연구기관, 과학기술분야, 정부출연금기관, 출자단체, 지방공사/공단)

#### 가. 대상 기관 분류 `국지공정과정출지`

| 구분 | 대상 기관 | 설명 |
|:-----|:---------|:-----|
| **국**가기관 | 중앙 정부 부처, 위원회 등 | 행정부, 입법부, 사법부 포함 |
| **지**방자치단체 | 시군구 지방자치단체 | 광역/기초 자치단체 |
| **공**공기관 | 「공공기관의 운영에 관한 법률」 제4조에 따른 공공기관 | 공기업, 준정부기관 등 |
| **정**부 최대지분 보유 기관 | 정부가 자본금을 출자하여 최대지분을 보유하고 있는 기관 또는 단체 | 단, 정부가 자본금의 2분의 1 미만을 출자한 금융기관은 제외 |
| **과**학기술분야 정부출연연구기관 | 「과학기술분야 정부출연 연구기관 등의 설립·운영 및 육성에 관한 법률」 제2조 | 한국과학기술연구원 등 |
| **정**부출연연구기관 | 「정부출연연구기관 등의 설립·운영 및 육성에 관한 법률」 제2조 | 한국개발연구원 등 |
| **출**연금 기관 또는 단체 | 법령에 따라 정부로부터 출연금을 지원받는 기관 또는 단체 | 정부 출연금 수혜 단체 |
| **지**방공사 또는 지방공단 | 「지방공기업법」에 따라 지방자치단체가 자본금의 2분의 1 이상을 출자한 지방공사 또는 지방공단 | 지역 공기업 |

> **법률 해석**: 「소프트웨어 진흥법」 시행령 제21조(국가기관등의 범위) 제4호의 '정부가 자본금을 출자하여...' 및 제6호의 '법령에 따라 정부로부터...'의 '정부'의 범위는 법제처의 법령해석의견을 존중하여 '지방자치단체'를 포함하는 것으로 해석

### 3. 소프트웨어사업 영향평가 체계

{: .highlight }
> **체계 구성**: 평가목표, 평가기준, 평가방법, 평가절차
> - (목표) `침예육` (침해예방, 예산낭비방지, 육성)
> - (기준) `유침필공` (유사성, 침해가능성, 필요성, 공공성)
> - (방법) `자과기` (자체평가, 과기정통부 검토, 기술지원)
> - (절차) `예발재` (예산편성, 발주준비, 재평가)

#### 나. 체계 구성요소 상세 `목기방절`

| 구성 요소 | 세부 요소 | 설명 |
|:---------|:---------|:-----|
| **평가 기준** | 평가 항목 | 민간 SW와의 유사성, 민간 SW시장 침해가능성, 영향평가 대상인 SW사업의 필요성·공공성 |
| | 평가 결과 | 민간 SW시장 침해 가능성 없음, 민간 시장 침해 최소화, 사업 재검토 |
| **평가 방법** | 자체 평가 | 국가기관등의 사업부서 또는 정보화부서에서 위 평가기준에 따라 자체적으로 평가하여 「소프트웨어사업 영향평가 결과서」 작성 |
| | 과기정통부 장관 검토 | 자체적으로 평가(재평가를 포함)하여 작성한 「소프트웨어사업 영향평가 결과서」를 과기정통부 장관에게 제출하여 검토받아야 함 (2023년 10월 19일 일부개정 시행) |
| | 기술 지원 | 국가기관등의 장은 SW사업 영향평가를 위하여 과기정통부 및 정보통신산업진흥원(NIPA)에 기술지원 요청 가능 |
| **평가 절차** | 예산 편성 단계 | 국가기관등의 장이 지능정보화사업 시행계획서 작성 시, 영향평가 실시 후 과기정통부 장관 검토 및 개선조치 요청 |

### 4. 소프트웨어사업 영향 평가 항목

{: .highlight }
> **평가 항목**: 민간 SW시장 침해가능성, SW사업의 필요성·공공성
> - (침해가능성) `유침` (유사성, 침해가능성)
> - (필요성·공공성) `법공사기` (법령규정, 공공데이터, 사업활성화, 기타사유)

#### 가. 평가 항목 상세 `침필공`

| 평가 항목 | 세부 요소 | 설명 |
|:---------|:---------|:-----|
| **민간 SW시장 침해가능성** | SW사업의 주요기능과 동일·유사한 SW를 민간에서 제공하는지 여부 | 민간 SW와의 유사성 분석 |
| | 영향평가 대상인 SW사업의 필요성·공공성 | 실제 사업 추진 시 사업의 필요성이나 공공성을 분석하여 작성 |
| **SW사업의 필요성·공공성** | 법령에 규정된 사업 | 국가기관등이 법령에 따라 SW사업을 추진하는 경우 주요 기능을 규정한 해당 관련 법령을 기술 |
| | 「공공데이터 활용 공공서비스 제공 및 정비 가이드라인」 준수 | 「공공데이터 활용 공공서비스 제공 및 정비 가이드라인」을 참고하여 구축하고자 하는 서비스가 정비 모델인지 정비 대상 검토 제외인지 확인 |
| | 사업을 통한 민간 서비스 시장 활성화 기여 | Open API 등을 통한 데이터 개방, 민간 SW 구매·활용 계획, 데이터 연계표준 및 표준업무 절차 제시, 중장기 민간 이양 계획 등 민간 SW어 시장 활성화 방안을 작성 |
| | 그 외의 사유로 민간이 서비스를 제공하기에 부적합 | 위의 사유 외에 민간이 서비스를 제공하기에 부적합한 사유를 상세하게 작성 |

### 5. 제도 개정 내용 (2023.10 개정)

{: .highlight }
> **주요 개정**: 과기정통부 장관 검토 의무화, 기한 구체화, 재평가 절차 명확화
> - (개정) `검기재` (검토의무화, 기한명확화, 재평가절차)

| 구분 | 개정 전 | 개정 후 |
|:-----|:-------|:-------|
| **예산 편성 단계** | 국가기관등의 장 SW사업 영향평가 실시, 실시 결과 과기정통부 장관에게 제출 | 국가기관등의 장 SW사업 영향평가 실시, 실시 결과 **과기정통부 장관에게 제출**, **과기정통부 장관 검토 및 개선조치 요청** |
| **발주 준비 단계** | 입찰공고 직전 국가기관등의 장이 영향평가 실시 | **입찰공고일 30일 전**(긴급한 경우 10일 전), 국가기관등의 장은 영향평가 실시 후 **과기정통부 장관에게 제출**, 과기정통부 장관은 **총 사업금액이 1억원 이상 신규 SW개발사업**에 대해 검토 및 개선조치 요청 |
| | 입찰공고 시 결과 공개 | **입찰공고일 5일 전**(긴급한 경우 3일 전)까지 결과 공개 |
| **재평가 요청 시** | SW사업자의 재평가 요청 근거는 존재, 재평가 기한에 대한 구체적 규정 부재 | 재평가는 국가기관등의 장이 **입찰공고 하기 전**까지 실시 |
| | SW사업자는 국가기관등의 장이 입찰공고시 공개한 결과서를 보고 재평가 신청, 국가기관등의 장은 이미 입찰공고가 개시된 시점으로 사업내용을 변경하는 것이 어려움 | SW사업자는 국가기관등의 장이 **입찰공고일 5일 전**까지(긴급한 경우 3일 전까지) 미리 공개한 결과서를 보고 재평가 신청, **국가기관등의 장은 재평가 결과를 반영**하여 SW사업 입찰 |
| | | 과기정통부 장관은 사업 **입찰공고 前** 국가기관등의 장이 수정한 재평가 결과 검토 및 개선조치 요청 |
