### 한 줄 정의

> **갈등관리**: 프로젝트 내·외부의 대립 상황을 적절히 조절하여 프로젝트 성과를 극대화하는 PM의 핵심 역량

### 쉬운 비유

| 갈등 부재 | 적정 수준 갈등 | 극한 갈등 |
|:---------|:-------------|:---------|
| 🌊 잔잔한 바다 - 배가 앞으로 안 나감 | ⛵ 적당한 바람 - 최적의 항해 속도 | 🌪️ 태풍 - 배가 뒤집어짐 |

### 왜 필요한가? (3줄 요약)

1. **적정 갈등 유지**: 갈등이 없으면 침체, 너무 많으면 분열 → 적정 수준 유지가 핵심
2. **팀 발달 이해**: 터크만 모델로 팀 단계별 갈등 특성과 리더십 스타일 파악
3. **해결 전략 선택**: 상황에 맞는 5가지 해결 전략(철회~문제해결)으로 효과적 대응

## 🎯 기술사 수준 설명

{: .highlight }
> **프로젝트 갈등관리**: 적정 수준 갈등, 해결 전략, 팀 발달 → PM 핵심 역량
> - (1) `부적극` (갈등과 성과 관계)
> - (2) `구인업환` + `철수타강문` (요인과 해결 전략)
> - (3) `FSNPA` (터크만 5단계)
> - ⭐ **차별점**: 역U자 곡선 - 적정 수준의 순기능적 갈등이 성과를 극대화

---

### 1. 갈등과 프로젝트 성과의 관계

{: .highlight }
> **갈등과 성과**: 역U자 곡선, 순기능/역기능, 적정 수준
> - (관계) `부적극`
> - (유형) `역순역`
> - (관점) `전인상`

#### 가. 갈등과 프로젝트 성과의 관계 개념도

```
   성과의      ▲
   크기        │         ╭──────╮
              │        ╱  적정   ╲
              │       ╱  수준의   ╲
              │      ╱   갈등     ╲
              │     ╱              ╲
              │    ╱                ╲  극한
              │   ╱                  ╲ 갈등
              │  ╱  갈등              ╲
              │ ╱   부재               ╲
              │╱                        ╲
   저 ────────┴─────────────────────────────▶
                    갈등의 정도              고
```

| 구분 | 정의 |
|:-----|:-----|
| **조직 갈등** | 조직 내의 개인 또는 집단 간에 목표, 이해관계, 가치 등의 차이로 인해 발생하는 대립 상황 |
| **프로젝트 성과** | 범위, 일정, 예산, 품질 등의 목표를 얼마나 효과적으로 달성했는지 평가하는 결과지표 |

#### 나. 갈등과 프로젝트 성과의 관계 상세 `부적극`

| 갈등 수준 | 갈등 유형 | 성과 | 특성 |
|:---------|:---------|:-----|:-----|
| **부**재 (A) | 역기능(Dysfunctional) | 낮음 | 적응력 둔화, 획일성, 무사안일, 포기, 침체 |
| **적**정 (B) | 순기능(Functional) | 높음 | 변화 지향, 창조적, 다양성, 도전적, 적극적 목표 수행 |
| **극**한 (C) | 역기능(Dysfunctional) | 낮음 | 혼란, 분열, 투쟁, 비협조, 불안, 위협, 목적의식 결여 |

#### 다. 갈등에 대한 관점 변화 `전인상`

| 관점 | 시각 | 접근법 |
|:-----|:-----|:-------|
| **전**통적 관점 | 부정적인 영향, 조직 내 역기능 | 갈등제거 (Conflict Resolution) |
| **인**간관계론적 관점 | 필연적 발생, 성과에 도움 가능 | 갈등수용 (Conflict Acceptance) |
| **상**호작용 관점 | 긍정적인 영향, 추진력, 효과 수행 | 갈등조장 (Conflict Stimulation) |

### 2. 갈등의 요인과 해결 전략

{: .highlight }
> **요인과 해결 전략**: 4가지 요인, 5가지 해결 방법
> - (요인) `구인업환`
> - (해결전략) `철수타강문`
> - (PMBOK) `일우자기관원대`

#### 가. 프로젝트 갈등의 요인

##### 그룹 1: 갈등 요인 분류 `구인업환`

| 구분 | 갈등 요인 | 설명 |
|:-----|:---------|:-----|
| **구**조적 요인 | 역할/책임 불명확성 | 구성원의 역할이나 책임이 명확하지 않을 때 발생 |
|  | 자원의 제한성 | 자금, 인력, 장비 등의 자원이 제한될 때 경쟁 유발 |
| **인**적 요인 | 가치/성격 차이 | 성격, 가치관, 태도의 차이로 인한 갈등 |
|  | 의사소통 오류 | 오해, 정보 부족 또는 왜곡된 전달로 갈등 유발 |
| **업**무 요인 | 목표 불일치 | 부서 또는 개인 간 목표가 충돌할 때 발생 |
|  | 상호의존성 증가 | 한 부서의 성과가 다른 부서에 의존될 때 책임공방 가능 |
| **환**경 요인 | 문화/정책 차이 | 부서나 팀마다 문화나 룰이 다를 경우 충돌 |
|  | 외부 규제 변화 | 시장, 규제, 기술 변화에 따른 내부 긴장 및 갈등 발생 |

##### 그룹 2: PMBOK 갈등 요인 `일우자기관원대`

| 요인 | 설명 |
|:-----|:-----|
| **일**정 | 일정에 대한 팀원 상호 간 동의(Commitment) 부족 시 갈등 발생 |
| 프로젝트 **우**선순위 | 자원할당, 위험관리 등에서 발생하는 프로젝트 우선순위의 이견 갈등 |
| **자**원 | 조직 차원만이 아닌 프로젝트에서도 우수한 혹은 한정된 인적자원을 서로 차지하려는 갈등 |
| **기**술적 옵션 | 기술적 방법이 다를 경우 갈등 발생 |
| **관**리 절차 | 관리 절차나 문서작업으로 인한 갈등 발생 |
| **원**가 | 프로젝트 경비 부족, 사용방법에 의한 갈등 발생 |
| **대**인 관계 | 팀원의 성격 차이에 의한 대인관계에서 갈등 발생 |

#### 나. 프로젝트 갈등의 해결 전략 `철수타강문`

```
 자기 의견    ▲
 관철 정도    │
   높음 ─────┼────────────────┬────────────────┐
             │   Forcing      │ Problem Solving│
             │   (강요)       │   (문제해결)   │
             │                │                │
   중간 ─────┼───────┬────────┼────────────────┤
             │       │Compromise              │
             │       │ (타협)  │              │
             │       │         │              │
   낮음 ─────┼───────┴─────────┼──────────────┤
             │ Withdrawing    │  Smoothing   │
             │  (철회)        │   (수용)     │
             └────────────────┴──────────────────▶
                  낮음        중간        높음
                        타 그룹에 대한 협조 정도
```

| 해결방법 | 특징 | 적용 상황 |
|:--------|:-----|:---------|
| **철**회 (Withdrawing) | 낮은 주장, 낮은 협력 | 이슈가 사소한 경우, 추가적인 정보가 필요한 경우, 자기 의견이 관철될 가능성이 매우 낮은 경우 |
| **수**용 (Smoothing) | 낮은 주장, 높은 협력 | 나중을 위하여 신용을 얻고자 하는 경우, 조화와 안정이 매우 중요한 경우(분위기가 우선일 때), 이슈가 갈등 상대방에게 보다 중요한 사안인 경우 |
| **타**협 (Compromising) | 중간 주장, 중간 협력 | 목표는 중요하나 더 이상 설득이 힘든 경우, 비슷한 파워를 가진 집단들끼리의 갈등인 경우, 양측이 어느정도 만족할 수 있는 합의점을 도출할 가능성이 존재하는 경우 |
| **강**요 (Forcing) | 높은 주장, 낮은 협력 | 인기 없는 정책이지만 꼭 필요한 정책을 집행할 때, 긴급한 사안을 결정해야 하는 경우, 갈등 상대방보다 경쟁우위에 있는 경우 |
| **문**제해결 (Problem Solving) | 높은 주장, 높은 협력 | 매우 중요한 통합된 의견을 도출하고자 하는 경우, 공감대를 형성해 지속적인 관계유지가 필요한 경우 |

### 3. 터크만(Tuckman)의 팀 발달 5단계 모델

{: .highlight }
> **터크만 모델**: Forming, Storming, Norming, Performing, Adjourning
> - (단계) `FSNPA` (형격규성해)
> - (리더십) `지지참위지`
> - (핵심) `Storming 극복이 Performing 진입의 핵심`

#### 가. 터크만(Tuckman)의 팀 발달 5단계 모델 개념도

```
   성과      ▲
             │                        Performing ④
             │                       ╱  (성과기)
             │                      ╱
             │    Forming ①       ╱ Norming ③        Adjourning ⑤
             │     (형성기)      ╱   (규범기)          (해산기)
             │         ╲        ╱
             │          ╲      ╱
             │           ╲    ╱
             │            ╲  ╱
             │     Storming ②
             │      (격동기)
             └─────────────────────────────────────────▶ 시간
```

| 구분 | 정의 |
|:-----|:-----|
| **터크만 모델** | 팀이 형성되고 해산하기까지의 과정을 성과에 따라 5단계로 제시한 팀 발달 모델 |
| **핵심 인사이트** | Storming 단계는 필수적이며, 해당 단계의 갈등 극복 및 Norming 단계의 전환 노력 필요 |

#### 나. 터크만의 5단계 팀 개발 모델 단계별 설명 `FSNPA`

| 단계 | Forming | Storming | Norming | Performing | Adjourning |
|:-----|:--------|:---------|:--------|:-----------|:-----------|
| **한글명** | 형성기 | 격동기 | 규범기 | 성과기 | 해산기 |
| **주요 관심** | 상호 인식 | 갈등 처리 | 협력 관계 구축 | 생산성 향상 | 과업 종료 |
| **과업 목표** | 열성 | 역할 명료화 | 몰입 | 성취 | 인도물 종료 |
| **관계상 목표** | 수용 | 소속감 | 지원 | 자긍심 | 팀관계 정리 |
| **주요 딜레마** | 불확실성 | 의견불일치 | 지원 vs 간섭 | 관심 vs 고립 | 상실감 |
| **필요 리더십** | 지시형 | 지도형 | 참여형 | 위임형 | 지시형 |
| **필요 활동** | 팀 방향 설정 | 계획 및 역할 명료화 | 업무 및 역할 몰입 | 수행관리 및 평가 | Lessons Learned |

#### 다. 단계별 PM의 역할과 활동

| 단계 | PM 역할 | 핵심 활동 |
|:-----|:--------|:---------|
| **Forming** | 지시적 리더십 | 수용적인 태도가 필요하며, 초반 단계로 리더의 지시적 리더십과 팀 방향 제시가 필요 |
| **Storming** | 지도형 리더십 | 팀원 내부의 소속감이 필요하며, 갈등 해결을 위한 지도형 리더십과 계획/역할 명확화가 필요 |
| **Norming** | 참여형 리더십 | 업무 및 역할에 몰입할 수 있도록 지원이 필요하며, 참여형 리더십과 몰입 목표 제시가 필요 |
| **Performing** | 위임형 리더십 | 성과 극대화를 위한 성취 목표와 위임형 리더십이 필요하며, 고립된 팀원이 없도록 자긍심을 고취시키는 것이 필요 |
| **Adjourning** | 지시형 리더십 | 과업 종료 단계로 팀관계를 정리하고 인도물을 종료하며, Lessons Learned의 관리가 필요 |

### ⭐ 차별점 키워드

| 항목 | 차별점 |
|:-----|:------|
| **성과 곡선** | 역U자 곡선 - 적정 수준의 순기능적(Functional) 갈등이 성과를 극대화 |
| **해결 전략** | 자기 주장 vs 타인 협력 2축 매트릭스로 5가지 전략 도출 |
| **터크만 모델** | Storming 단계 극복이 Performing 진입의 핵심 - 갈등은 팀 발달의 필수 과정 |
| **리더십 변화** | 단계별 리더십 전환: 지시형 → 지도형 → 참여형 → 위임형 → 지시형 |
