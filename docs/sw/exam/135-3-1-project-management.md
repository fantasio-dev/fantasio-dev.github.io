### 한 줄 정의

> **IT 프로젝트 관리**: IT 프로젝트의 목표를 달성하기 위해 계획, 실행, 모니터링, 통제 및 종료를 포함하는 전체 생명주기를 관리하는 체계적 관리방법론

### 쉬운 비유

| 프로젝트 | 프로그램 | 포트폴리오 |
|:--------|:--------|:----------|
| 🏠 집 한 채 짓기 | 🏘️ 아파트 단지 개발 (여러 동) | 🏙️ 신도시 전체 건설 (여러 단지) |

### 왜 필요한가? (3줄 요약)

1. **목표 달성**: 일정, 예산, 품질 목표를 체계적으로 관리하여 프로젝트 성공률 향상
2. **리스크 관리**: 발생 가능한 위험을 사전에 식별하고 대응하여 프로젝트 실패 방지
3. **자원 최적화**: 인력, 예산, 시간 등 한정된 자원을 효율적으로 배분하여 ROI 극대화

### 1. IT 프로젝트 관리의 개념

{: .highlight }
> **IT 프로젝트 관리 개념**: ISO21500, PMBOK, 계획-실행-통제
> - (표준) `ISO21500`, `PMBOK`
> - (지식영역) `통이범자시원리품조의`
> - (특징) `한시적`, `고유성`, `점진적 상세화`

#### 가. IT 프로젝트 관리의 개념도

```
┌─────────────────────────────────────────────────────────────────────┐
│                     IT 프로젝트 관리 프레임워크                      │
├─────────────────────────────────────────────────────────────────────┤
│                           ┌───────┐                                 │
│                           │이해   │                                 │
│         ┌─────┐          │관계자 │          ┌─────┐               │
│         │통합 │◀─────────┤       ├─────────▶│범위 │──────▶ 자원   │
│         └──┬──┘          └───┬───┘          └─────┘                │
│            │                 │                                      │
│   ┌────────┼────────────────┼────────────────────┐                 │
│   │        ▼                ▼                    │                 │
│   │  ┌──────────────────────────────────────┐   │                 │
│ 의│  │         착수 ──▶ 기획                │   │시               │
│ 사│  │            │         │               │   │간               │
│ 소│  │          통제 ◀──── 이행             │   │                 │
│ 통│  │            │                          │   │                 │
│   │  │          종료                         │   │                 │
│   │  └──────────────────────────────────────┘   │                 │
│   └──────────────────────────────────────────────┘                 │
│         조달 ◀──── 품질 ◀──── 리스크 ◀──── 원가                   │
└─────────────────────────────────────────────────────────────────────┘
```

| 구분 | 정의 |
|:-----|:-----|
| **IT 프로젝트 관리** | IT 프로젝트를 성공적으로 완수하기 위해 프로젝트의 계획, 실행, 모니터링, 통제 및 종료를 포함하는 전체 생명주기를 관리하는 체계적 관리방법론 |
| **관련 표준** | ISO 21500 (De-jure), PMBOK (De-facto) |

#### 나. 프로젝트 관리 지식영역 `통이범자시원리품조의`

| 주제그룹 | 개념 | 주요 산출물 |
|:--------|:-----|:-----------|
| **통**합 | 프로젝트와 관련된 다양한 활동과 프로세스를 도출, 정의 결합, 단일화, 조정, 통제, 종료에 필요한 프로세스 | 프로젝트 헌장, 프로젝트관리 계획서 |
| **이**해관계자 | 프로젝트 스폰서, 고객사, 기타 이해관계자를 식별하고 관리 | 이해관계자 등록대장 |
| **범**위 | 작업과 인도물을 식별하고 정의 | 작업분류체계 (WBS) |
| **자**원 | 인력, 시설, 장비, 자재, 기반 기설, 도구와 같은 적절한 프로젝트 자원을 식별하고 확보 | 자원계획, 프로젝트 조직도 |
| **시**간 | 프로젝트 활동의 일정을 수립하고 일정 통제의 진척상황을 관찰 | 활동기간산정, 일정표 |
| **원**가 | 예산 개발과 원가통제의 진척상황을 관찰 | 원가산정, 예산 |
| **리**스크 | 위험과 기회를 식별하고 관리 | 리스크 관리 대장 |
| **품**질 | 품질보증과 품질통제를 계획하고 확립 | 품질계획, 시정조치 |
| **조**달 | 계획에 요구된 프로세스를 포함하여 제품 및 서비스 또는 인도물을 인수하고 공급자와의 관계를 관리 | 조달 계획, RFI, RFP |
| **의**사소통 | 프로젝트와 관련된 정보를 계획, 관리, 배포 | 의사소통 계획, 정보배포 |

### 2. IT 프로젝트 관리 프로세스

{: .highlight }
> **프로세스 그룹**: 착수, 계획, 실행, 감시 및 통제, 종료
> - (그룹) `착계실감종`
> - (착수) `이관헌`
> - (종료) `인교`

#### 가. IT 프로젝트 관리 프로세스의 구성도

```
                        프로젝트 승인
                            │
    ┌───────────────────────▼───────────────────────┐
    │                                               │
    ▼           변경              계획              ▼
┌──────┐  ◀────────────  ┌──────┐  ─────────▶  ┌──────┐
│ 착수 │                 │ 기획 │               │ 실행 │
└──────┘                 └──────┘               └──────┘
    │                       ▲                      │
    │                       │     실적             │
    │                       │   시정조치           │
    │                    ┌──┴───┐                  │
    │                    │ 통제 │ ◀────────────────┘
    │                    └──┬───┘
    │                       │
    │                    인도물
    │                       │
    │                    ┌──▼───┐
    └──────────────────▶ │ 종료 │
                         └──────┘
```

#### 나. 프로세스 그룹의 상세 설명 `착계실감종`

| 프로세스 | 주요 활동 | 설명 |
|:--------|:---------|:-----|
| **착**수 | 이해관계자 식별 | 프로젝트의 주요 요구사항과 관련된 이해관계자 식별 |
|  | 프로젝트 관리자 임명 | 원활한 진행을 위한 관리자의 배정 및 진행준비 임무부여 |
|  | 프로젝트 헌장 작성 | 헌장 작성을 프로젝트 관리자에 공식적인 관리권한을 부여 |
| **계**획 | 프로젝트 관리 계획서 작성 | 모든 보조적인 계획서를 정의하고 작성 및 조율하여 하나의 종합적인 관리계획서에 통합한 문서 |
|  | 기준선 작성 | 프로젝트 생명주기 동안 프로젝트 진행률 관리 및 성과 측정을 위한 기준이 되는 라인 |
| **실**행 | 정의된 작업 수행 | 관리 계획에서 정의한 작업을 완료하기 위한 프로세스로 구성 |
|  | 변경사항에 따른 조정작업 | 실행 기간에 변경사항이 적용되면, 관리 계획서 갱신 작업과 기준선 변경 및 조정 작업이 수반됨 |
|  | 인도물 작성 | 프로젝트 작업 완료를 목표로 하는 인도물 작성 |
| **감**시 및 **통**제 | 조치 수행 | 발생가능성이 있는 위험 및 이슈에 대한 예방조치와 시정조치 수행 |
|  | 프로젝트 진행률 확인 | 현재 프로젝트의 진행률을 계획서의 일정 및 원가 기준선을 맞추고 있는지 확인하고 예측함 |
|  | 변경사항 승인 | 범위 변경사항에 대한 승인을 통하여 실행단계를 통해 수행할 수 있도록 승인 작업 및 검토 수행 |
| **종**료 | 프로젝트 인수 진행 | 프로젝트 고객과 스폰서에게 프로젝트를 공식적으로 인수요청 |
|  | 습득한 교훈물 | 프로젝트 기간동안 얻은 교훈 및 좋은 사례를 다음 프로젝트에 참조할 수 있도록 산출물을 정리함 (Lessons Learned) |

| 주제그룹 | 착수 | 기획 | 실행 | 통제 | 종료 |

### 3. IT 프로젝트 관리, 프로그램 관리, 포트폴리오 관리의 비교

{: .highlight }
> **3가지 관리 비교**: 프로젝트 < 프로그램 < 포트폴리오
> - (계층) `프<프<포`
> - (비교) `목범관위성감`
> - (포트폴리오) `전략적 투자 최적화`

#### 가. IT 프로젝트 관리, 프로그램 관리, 포트폴리오 관리의 개념 비교

```
┌─────────────────────────────────────────────────────────────────────┐
│                         포트폴리오                                   │
│  ┌────────────────────────────────────────────────────────────┐    │
│  │                     프로그램                                │    │
│  │  ┌──────────┐  ┌──────────┐  ┌──────────┐                 │    │
│  │  │프로젝트 1│  │프로젝트 2│  │프로젝트 n│   + 기타활동    │    │
│  │  └──────────┘  └──────────┘  └──────────┘                 │    │
│  └────────────────────────────────────────────────────────────┘    │
│                                                                     │
│  ┌────────────────────────────────────────────────────────────┐    │
│  │                     프로그램                                │    │
│  │  ┌──────────┐  ┌──────────┐  ┌──────────┐                 │    │
│  │  │프로젝트  │  │프로젝트  │  │프로젝트  │   + 기타활동    │    │
│  │  └──────────┘  └──────────┘  └──────────┘                 │    │
│  └────────────────────────────────────────────────────────────┘    │
└─────────────────────────────────────────────────────────────────────┘
```

| 구분 | IT 프로젝트 관리 | 프로그램 관리 | 포트폴리오 관리 |
|:-----|:----------------|:-------------|:---------------|
| **정의** | 특정 목표를 가진 개별 IT 프로젝트를 계획, 실행 및 통제하는 관리 방식 | 연관된 여러 프로젝트를 통합적으로 관리하여 시너지 효과를 극대화하는 관리 방식 | 조직의 전략적 목표와 일치하도록 프로젝트와 프로그램을 선정하고 관리하는 방식 |

#### 나. IT 프로젝트 관리, 프로그램 관리, 포트폴리오 관리의 상세 비교 `목범관위성감`

| 구분 | IT 프로젝트 관리 | 프로그램 관리 | 포트폴리오 관리 |
|:-----|:----------------|:-------------|:---------------|
| **목**적 | 프로젝트 목표 달성 (예: SW 개발, 시스템 구축 등) | 프로젝트 간 조정 및 효율적인 자원 활용 | 기업의 비즈니스 목표와 연계된 최적의 투자 및 가치 극대화 |
| **범**위 | 단일 프로젝트 | 여러 개의 관련 프로젝트 그룹 | 조직 전체의 프로젝트 및 프로그램 집합 |
| **관**리 요소 | 일정, 예산, 품질, 위험, 성과 등 | 프로젝트 간 가용자원, 우선순위, 리스크 | 프로젝트 및 프로그램의 투자 우선순위, 전략적 방향 |
| **위**험관리 | 개별 프로젝트의 리스크 식별, 및 대응 | 프로젝트 간 리스크 조정 및 영향 최소화 | 조직 전체의 리스크 식별 및 포트폴리오 균형 유지 |
| **성**과 측정 | 프로젝트 목표 달성여부 (예: 일정 준수, 예산 내 완료, 요구사항 충족) | 프로젝트 간 상호작용과 시너지 효과 극대화 여부 | 비즈니스 목표 달성 기여도 및 투자 대비 성과 |
| **감**시 | 프로젝트 관리자는 제품, 서비스, 프로젝트 결과에 대해 감시 및 통제 | 프로그램 관리자는 목표, 일정, 예산, 이익을 프로그램 컴포넌트별 감시 | 포트폴리오 관리자는 전략적인 변경 및 성과에 대하여 감시 |
| **예시** | 특정 앱 개발 프로젝트, 네트워크 구축 프로젝트 | 하나의 플랫폼 내에서 앱 개발, 데이터 분석, 보안 프로젝트를 통합 관리 | 기업 전체의 IT 프로젝트 투자 전략 및 자원 배분 최적화 |

### ⭐ 차별점 키워드

| 항목 | 차별점 |
|:-----|:------|
| **계층 관계** | 프로젝트 < 프로그램 < 포트폴리오 (규모와 전략적 수준 증가) |
| **프로젝트** | 한시적(Temporary), 고유성(Unique), 점진적 상세화(Progressive Elaboration) |
| **프로그램** | 여러 프로젝트의 시너지 효과 극대화 |
| **포트폴리오** | 조직의 전략적 목표와 연계된 투자 최적화 |
| **표준** | De-jure: ISO 21500 / De-facto: PMBOK |

### 📈 상위 토픽 계층도

```
프로젝트 관리
├── IT 프로젝트 관리  ← 현재 토픽
│   ├── 프로세스 그룹 (착계실감종)
│   │   ├── 착수
│   │   ├── 계획
│   │   ├── 실행
│   │   ├── 감시 및 통제
│   │   └── 종료
│   └── 지식영역 (통이범자시원리품조의)
│       ├── 통합, 이해관계자, 범위, 자원
│       └── 시간, 원가, 리스크, 품질, 조달, 의사소통
├── 프로그램 관리
│   └── 여러 프로젝트 통합 관리
└── 포트폴리오 관리
    └── 전략적 투자 최적화
```
