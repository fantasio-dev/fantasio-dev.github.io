### 🔑 핵심 키워드 3개

| 키워드 | 설명 |
|:------|:-----|
| **역공학** | 완성된 제품을 분해해서 설계도를 알아내는 것 |
| **재공학** | 기존 시스템을 분석해서 더 좋게 다시 만드는 것 |
| **3R** | Reverse(역공학), Restructure(재구조화), Re-engineering(재공학) |

---

### 📖 등장배경

```
🌐 과거: 오래된 시스템(레거시)이 많아졌는데, 설계 문서가 없거나 개발자가 퇴사함

💡 문제: "문서도 없고 개발자도 없는데, 이 시스템을 어떻게 유지보수하지?"

✅ 해결: 역공학과 재공학으로 해결!
   → 역공학: 코드를 분석해서 설계 문서 복원
   → 재공학: 복원된 정보로 시스템 현대화
   → 유지보수 비용 절감, 시스템 수명 연장
```

---

### 📝 개념 정의

> **역공학**: 완성품을 분해해서 설계를 알아내는 과정
> **재공학**: 기존 시스템을 개선해서 다시 만드는 과정

### 쉬운 비유

| 역공학 | 재공학 |
|:------|:------|
| 🔧 완성된 자동차를 분해해서 설계도 복원 | 🚗 낡은 자동차를 최신 엔진으로 교체 |
| 레시피 없이 요리 맛보고 재료 알아내기 | 알아낸 레시피로 더 맛있게 개선하기 |

---

### 🏗️ 기술요소

#### 그룹 1: 역공학 프로세스 `추분문논`

| 항목 | 설명 | 예시 |
|:-----|:-----|:-----|
| **추**출 | 실행파일에서 소스코드 추출 | JD-GUI, Ollydbg |
| **분**석 | 정적/동적 분석으로 코드 이해 | WireShark, IDA Pro |
| **문**서화 | 분석 결과를 설계 문서로 작성 | UML, ERD |
| **논**리/자료 역공학 | 논리 구조와 데이터 구조 복원 | 물리적 설계정보 추출 |

#### 그룹 2: 재공학 프로세스 `역재구모`

| 항목 | 설명 | 예시 |
|:-----|:-----|:-----|
| **역**공학 | 기존 시스템 분석 및 문서 추출 | 소스코드, 설계문서 추출 |
| **재**구조화 | 프로그램 구조와 데이터 재구조화 | 의미론적 정보 추출 |
| **구**현 | 개선된 시스템 빌드 | 성능 개선된 Target System |
| **모**듈화 | 시스템 모듈 구조 변경 | 클러스터 분석, 결합도 개선 |

---

### ⭐ 차별점 키워드

**역공학 vs 재공학**: 역공학은 **분석/이해** 목적, 재공학은 **개선/변환** 목적

---

### 📋 6하원칙 요약

| 구분 | 역공학 | 재공학 |
|:-----|:------|:------|
| **What** | 기존 시스템 분석/문서화 | 기존 시스템 개선/현대화 |
| **Why** | 문서 없는 레거시 이해 | 유지보수성 향상 |
| **Who** | 개발자, 분석가 | 개발팀, 아키텍트 |
| **When** | 레거시 시스템 인수인계 시 | 시스템 현대화 필요 시 |
| **Where** | 소스코드, 실행파일 | 전체 시스템 |
| **How** | 디컴파일, 정적/동적 분석 | CASE 도구, 재구조화 |

## 🎯 기술사 수준 설명

{: .highlight }
> **역공학/재공학**: 레거시 시스템 문서화 필요 → 역공학, 재공학, 3R → SW 유지보수성 향상 기법
> - (역공학) `추분문` (추출, 분석, 문서화)
> - (재공학) `역재구` (역공학, 재구조화, 구현)
> - (적용방법) `재재의` (재구조화, 재모듈화, 의미론적 정보추출)
> - ⭐ **차별점**: 역공학은 분석/이해 목적, 재공학은 개선/변환 목적

---

### 📝 개념 정의

| 구분 | 정의 |
|:-----|:-----|
| **역공학** | SW 디버깅/디컴파일로 구조·원리·알고리즘을 역분석하여 재구성하는 과정 |
| **재공학** | 자동화 도구로 현존 시스템을 점검·수정하여 순공학을 준비하는 과정 |
| **3R** | Reverse Engineering, Restructure, Re-engineering의 통합 개념 |

---

### 🏗️ 구성요소

#### 그룹 1: 역공학 프로세스 `추분문`

| 항목 | 설명 | 예시 |
|:-----|:-----|:-----|
| **추**출 (Code 추출) | Ollydbg, JD-GUI 등으로 Dirty Code 추출 | 디컴파일러, 디버거 |
| **분**석 (Code 분석/수정) | 정적/동적 분석, Packet/Log 분석으로 Clean Code化 | WireShark, IDA Pro |
| **문**서화 | 분석된 코드와 도메인 지식으로 명세서 작성 | UML, ERD, 설계문서 |
| **논**리/자료 역공학 | 물리적 설계정보 추출, DB 전이 | 데이터 구조 복원 |

#### 그룹 2: 재공학 프로세스 `역재구모`

| 항목 | 설명 | 예시 |
|:-----|:-----|:-----|
| **역**공학 (Reverse Eng.) | Source Code, 설계문서 추출, PE 구조 분석 | Assembly Code 추출 |
| **재**구조화 | 의미론적 정보 추출, Program/Data 구조 재구조화 | 코드 리팩토링 |
| **구**현 | 재구조화된 Program+Data 합성, 성능 개선 시스템 Build | 새 시스템 배포 |
| **모**듈화 | 시스템 모듈 구조 변경, 클러스터 분석/결합도 관련 | CASE 도구 활용 |

### 🔄 순공학-역공학-재공학 관계도

```
┌─────────────────────────────────────────────────────────────────┐
│                  순공학 → 역공학 → 재공학 관계                    │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   ┌─────────┐        순공학         ┌─────────┐                 │
│   │ 요구사항 │ ─────────────────────▶│ 시스템  │                 │
│   │  분석   │    (Forward Eng.)     │  구현   │                 │
│   └─────────┘                       └────┬────┘                 │
│        ▲                                 │                      │
│        │                                 │ 역공학               │
│        │         ┌─────────┐             │ (Reverse Eng.)       │
│        │         │  재공학  │◀────────────┘                      │
│        │         │ (Re-Eng)│                                    │
│        │         └────┬────┘                                    │
│        │              │                                         │
│        └──────────────┘                                         │
│            재설계/개선                                           │
│                                                                 │
│   ※ 순공학: 요구사항 → 설계 → 구현                               │
│   ※ 역공학: 구현 → 설계 → 요구사항 (역방향 분석)                  │
│   ※ 재공학: 역공학 결과를 바탕으로 시스템 개선                     │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

---

### ⭐ 차별점 키워드

| 키워드 | 설명 |
|:------|:-----|
| **분석 vs 개선** | 역공학은 기존 시스템 분석/이해, 재공학은 시스템 개선/변환 |
| **방향성** | 역공학은 구현→설계 (역방향), 재공학은 분석→재설계→구현 (순방향) |
| **3R 연계** | Reverse → Restructure → Re-engineering 순환 구조 |
| **유지보수성** | 레거시 시스템의 유지보수성 향상이 궁극적 목적 |

