## 🎯 기술사 수준 설명

### 📌 핵심 암기 (Quick Reference)

{: .highlight }
> **역공학/재공학**: 유지보수, 3R(Reverse, Re-structure, Re-engineering) → SW 생산성 및 유지보수성 향상
> - (역공학) `추분문` (Code 추출, 분석/수정, 문서화)
> - (재공학) `역재구` (역공학, 재구조화, 구현)
> - ⭐ **차별점**: 역공학은 분석/이해, 재공학은 개선/변환 목적

---

## 핵심 키워드

`역공학(Reverse Engineering)` `재공학(Re-engineering)` `3R` `유지보수` `CASE 도구`

---

## 정의/개념

| 구분 | 정의 |
|:-----|:-----|
| **역공학** | SW에 대한 디버깅, 디컴파일 등의 분석을 통해 기존 제품에 구현된 구조, 원리, 기술, 알고리즘 등을 역으로 분석하여 재구성하는 과정 |
| **재공학** | 자동화된 도구로 현존하는 시스템을 점검 또는 수정하는 프로세스로 시스템의 재설계, 교체를 CASE 도구로 사용하도록 순공학을 준비하는 과정 |

---

```
│                                                                 │

```

## 역공학 프로세스 `추분문`

| 암기 | 단계 | 설명 | 도구/기법 |
|:-----|:-----|:-----|:---------|
| **추** | Code 추출 | Ollydbg, JD-GUI 등의 Tool 사용, Dirty Code 추출 | 디컴파일러, 디버거 |
| **분** | Code 분석/수정 | 정적/동적 분석 및 Clean Code化, Packet/Log 분석 | WireShark, IDA Pro |
| **문** | 문서화 | 분석된 Source Code와 도메인 지식으로 명세서 작성 | UML, ERD |

### 역공학 종류

| 종류 | 설명 |
|:-----|:-----|
| **논리 역공학** | 원시 코드로부터 정보를 추출하여 물리적 설계 정보 저장소에 저장, 물리적 설계정보를 얻어 내는 수단 |
| **자료 역공학** | 기존 데이터베이스를 수정하거나 새로운 DBMS로 전이하는 역할 |

---

## 재공학 프로세스 `역재구`

| 암기 | 단계 | 설명 |
|:-----|:-----|:-----|
| **역** | Reverse Engineering | Source Code, 설계 문서 추출, PE 구조 분석, Assembly Code 추출 |
| **재** | 재구조화 | 의미론적 정보 추출, Program 구조와 데이터 재구조화 |
| **구** | 구현 | 재구조화된 Program과 Data를 합성하여 성능 개선된 시스템 Build |

### 재공학 적용 방법

| 방법 | 설명 |
|:-----|:-----|
| **재구조화** | SW 부품을 라이브러리에 모아 놓고 새로운 SW 개발에 필요한 부품을 찾아내어 결합 |
| **재모듈화** | 시스템의 모듈 구조를 변화시키는 것으로 시스템 구성요소의 클러스터 분석 및 결합도와 관련 |
| **의미론적 정보 추출** | 코드 수준이 아닌 문서 수준의 복구 방법 |

---

## 역공학 vs 재공학 비교

| 비교 항목 | 역공학 (Reverse Eng.) | 재공학 (Re-engineering) |
|:---------|:---------------------|:----------------------|
| **목적** | 기존 시스템 분석/이해 | 기존 시스템 개선/변환 |
| **방향** | 구현 → 설계 → 요구사항 | 분석 → 재설계 → 구현 |
| **산출물** | 설계 문서, 명세서 | 개선된 시스템 |
| **도구** | 디컴파일러, 디버거 | CASE 도구 |
| **활용** | 레거시 시스템 분석, 보안 취약점 분석 | 시스템 현대화, 마이그레이션 |
