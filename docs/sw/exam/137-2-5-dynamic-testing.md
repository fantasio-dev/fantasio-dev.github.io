### 1. SW 실행 기반 검증 기법, 동적 테스트의 개요

{: .highlight }
> **동적 테스트**: SW 실행, 실제 동작 검증, 정상 동작 여부 확인
> - (정의) `실행하면서 검증`
> - (특징) `개기보ISO` (개발후반, 기능오류, 보안취약점, ISO 29119-4)
> - (유형) `명구` (명세기반, 구조기반)

#### 가. 개념 정의

| 구분 | 내용 |
|:-----|:-----|
| **정의** | 소프트웨어를 실행하여 실제 동작과 결과를 검증하는 테스트 기법 |
| **대상** | 실행파일, 배포된 SW 등 실제 프로그램을 실행하면서 검증 |
| **목적** | 개발 프로세스에 따라 구현된 프로그램을 실행하면서 테스트 기준에 부합 여부 및 결함 확인 |

#### 나. 동적 테스트 분류 개념도

```
                        ┌─────────────────┐
                        │    SW 테스트     │
                        └────────┬────────┘
                                 │
              ┌──────────────────┴──────────────────┐
              │                                     │
              ▼                                     ▼
      ┌───────────────┐                     ┌───────────────┐
      │   SW 실행     │                     │   SW 미실행    │
      │  (동적 테스트) │                     │  (정적 테스트) │
      └───────┬───────┘                     └───────┬───────┘
              │                                     │
      ┌───────┴───────┐                     ┌───────┴───────┐
      │               │                     │               │
      ▼               ▼                     ▼               ▼
┌───────────┐   ┌───────────┐         ┌─────────┐   ┌─────────────┐
│ 명세 기반  │   │ 구조 기반  │         │ Review  │   │ Walk Through│
│  테스트   │   │  테스트   │         └─────────┘   └─────────────┘
└───────────┘   └───────────┘               ┌─────────────┐
                                            │ Inspection  │
                                            └─────────────┘
```

#### 다. 동적 테스트 특징

| 특징 | 설명 |
|:-----|:-----|
| **개**발 후반 수행 | 개발 후반 및 테스트 단계에서 수행 |
| **기**능오류 검출 | 기능오류, 성능문제, 보안취약점 검출 |
| **보**안취약점 검출 | 실제 실행 환경에서 보안 취약점 발견 |
| **ISO**/IEC 29119-4 포함 | 국제 표준 기반 체계적 테스트 기법 |

### 2. 동적 테스트의 명세 기반 테스트와 구조 기반 테스트 비교

{: .highlight }
> **명세 vs 구조 비교**: 문서 기반 vs 코드 기반, 블랙박스 vs 화이트박스
> - (명세) `동분경의상유페` (동등분할, 분류트리, 경계값, 의사결정, 상태전이, 유즈케이스, 페어와이즈)
> - (구조) `제루구결조조변다` (제어구조, Loop, 구문, 결정, 조건, 조건결정, 변경조건결정, 다중조건결정)

#### 가. 개념 측면 비교

| 구분 | 명세 기반 테스트 | 구조 기반 테스트 |
|:-----|:--------------|:--------------|
| **개념** | 요구사항 및 기능 명세를 기반으로 외부 동작을 검증하는 테스트 (Black-box) | 소스코드 및 제어 흐름을 기반으로 내부 로직을 검증하는 테스트 (White-box) |
| **장점** | ① 요구사항 추적성 확보, 변경 영향 분석 ② 테스트 초기단계 적용 용이 | ① 결함 위치 파악 용이, 코드 수준 분석 ② 커버리지 기반 품질 측정 지표 제공 |

#### 나. 세부 특성 비교 `목정자기관대 장단`

| 구분 | 명세 기반 테스트 | 구조 기반 테스트 |
|:-----|:--------------|:--------------|
| **목**적 | 기능 검증 (요구사항 충족 여부 확인) | 논리 검증 (코드 로직 및 흐름 결함 탐지) |
| **정**보 활용 | 명세서, 요구사항, 설계 문서 | 소스코드, 제어 흐름, 데이터 흐름 |
| **자**동화 가능성 | 가능 (테스트 데이터 자동 생성 도구 활용) | 가능 (커버리지 측정 및 자동화 도구 활용) |
| 도출 **기**법 | 동등 분할, 경계값 분석, 결정 테이블, 상태 전이, 분류 트리 | 문장, 분기, 조건, 경로 커버리지 |
| 테스트 **관**점 | 외부 관점 (사용자 관점) | 내부 관점 (개발자 관점) |
| 테스트 **대**상 | 기능, 인터페이스, 요구사항 | 코드 로직, 제어 구조 |
| **장**점 | 요구사항 누락 발견, 사용자 시나리오 적합 | 로직 오류 탐지, 커버리지 측정 용이 |
| **단**점 | 내부 로직 오류 탐지 어려움 | 요구사항 누락 검증 어려움 |
| 대표 사례 | UI 테스트, 기능 테스트, 인수 테스트 | 단위 테스트, 보안 취약점 분석, 리팩토링 검증 |
| 테스트 단계 | 시스템 테스트, 인수 테스트 | 단위 테스트, 통합 테스트 |
| 커버리지 지표 | 요구사항/기능 커버리지 | 문장/분기/조건/경로 커버리지 |
| 주요활용 산업 | 금융, 공공, 사용자 중심 시스템 | 임베디드, 보안, 미션 크리티컬 시스템 |

> 명세 기반 테스트는 요구사항 중심의 외부 동작 검증, 구조 기반 테스트는 코드 중심의 내부 논리 검증으로 상호 보완적으로 활용

### 3. 주어진 프로그램 명세의 동등 분할과 분류 트리 기법 테스트 케이스 설명

{: .highlight }
> **테스트 케이스 작성**: 동등 분할 (입력 영역 분할) + 분류 트리 (트리 구조 시각화)
> - (동등 분할) `입대테` (입출력 영역 분할, 대표값 선정, 테스트 케이스 도출)
> - (분류 트리) `선설생` (대상 항목 선정, 분류 트리 설계, 테스트 케이스 생성)

#### 가. 프로그램 명세

```
[프로그램 명세]
이 기능의 입력청은 정수로 입력되며, 입력값이 10 미만이거나 100을 초과하면 "Fail", 
10이상이고 100이하면 "Pass", 정수가 아니면 "Invalid Input" 메시지를 출력한다.
```

#### 나. 동등 분할 기법 테스트 케이스

**1) 동등 분할 개념**

| 구분 | 내용 |
|:-----|:-----|
| **개념** | 데이터의 구간별 대표 값 도출, 테스트 기법 |
| **원리** | 입력값을 동등 클래스(Equivalence Class)로 분할하여 각 클래스를 대표하는 테스트 케이스를 도출 |

**2) 동등 클래스 분류**

```
              입력범위
     ◀────────────────────────────────▶
     │                                │
     0        40        80       120
     │         │         │         │
     ▼         ▼         ▼         ▼
 ┌───────┐ ┌───────┐ ┌───────┐ ┌───────┐
 │ 대표값 │ │ 대표값 │ │ 대표값 │ │ 대표값 │
 │ EC1   │ │ EC2   │ │ EC3   │ │ EC4   │
 └───────┘ └───────┘ └───────┘ └───────┘
```

| 식별 | 동등 클래스 (범주) | 유형 | 기대 결과 |
|:---:|:-----------------|:----:|:---------|
| EC1 | 비정수 | 무효 | Invalid Input |
| EC2 | 정수 < 10 | 무효 | Fail |
| EC3 | 정수 10~100 (포함) | 유효 | Pass |
| EC4 | 정수 > 100 | 무효 | Fail |

**3) 대표값 선정 및 테스트 케이스 작성**

- 설계방식은 One-To-One: 클래스별 1개 지정
  - EC1: "12.5" (비정수 대표)
  - EC2: 9 (하한 아래)
  - EC3: 55 (정상 구간의 중앙값 대표)
  - EC4: 101 (상한 위)

| TC ID | 입력값 | 테스트 구간 | 기대 결과 | 목적 |
|:-----:|:-----:|:----------|:---------|:-----|
| TC1 | -20.1 | 입력값 < 0 | Invalid Input | 비정상 입력값 확인 |
| TC2 | -20 | 입력값 < 0 | Fail | 정상 범위 구간 밖 입력값 확인 |
| TC3 | 20 | 0 <= 입력값 < 40 | Pass | 정상 범위 구간 내 입력값 확인 |
| TC4 | 60.1 | 40 <= 입력값 < 80 | Invalid Input | 비정상 입력값 확인 |
| TC5 | 100 | 80 <= 입력값 < 120 | Pass | 정상 범위 구간 내 입력값 확인 |
| TC6 | 140 | 120 <= 입력값 | Fail | 정상 범위 구간 밖 입력값 확인 |
| TC7 | 140.1 | 120 <= 입력값 | Invalid Input | 비정상 입력값 확인 |

> 입력 구간을 동등 분할하고 대표값을 선정하여 테스트 진행 및 결과 확인

#### 다. 분류 트리 기법 테스트 케이스

**1) 분류 트리 개념**

| 구분 | 내용 |
|:-----|:-----|
| **개념** | 흐름을 트리구조로 시각화 하여 케이스 설계하고 입력 변수 간 조합, 경험 기반/경계값 분석 등 활용 케이스 도출 |
| **원리** | 테스트 대상 시스템의 입력조건을 분류, 트리구조로 표현하여 테스트 케이스 생성 |

**2) 분류 트리 작성 절차** `선설생`

| 단계 | 절차 | 설명 |
|:---:|:-----|:-----|
| 1 | 테스트 대상 항목 **선**정 | 테스트 대상 시스템을 독립성을 띠는 (개별적으로 테스트가 가능한) 여러 개의 테스트 대상 항목(test objects) 분할 |
| 2 | 분류 트리 **설**계 | 이전 단계에서 분할된 각 대상 항목에 대해서 필요한 시 되는 모든 측면을 고려하여 분류 트리 구축 |
| 3 | 테스트 케이스 **생**성 | 분류 클래스들을 조합하여 테스트 대상 항목의 가장 중요한 테스트 조건이 커버되도록 테스트 케이스 도출 |

**3) 분류 트리 구조도**

```
                    ┌─────────────┐
                    │   입력값     │
                    └──────┬──────┘
                           │
           ┌───────────────┴───────────────┐
           │                               │
           ▼                               ▼
     ┌───────────┐                   ┌───────────┐
     │   비정수   │                   │    정수    │
     └─────┬─────┘                   └─────┬─────┘
           │                               │
           │                   ┌───────────┼───────────┐
           │                   │           │           │
           ▼                   ▼           ▼           ▼
                          ┌────────┐  ┌────────┐  ┌────────┐
                          │10 미만 │  │10 이상  │  │100 초과│
                          │        │  │100 이하│  │        │
                          └───┬────┘  └───┬────┘  └───┬────┘
                              │           │           │
                              ▼           ▼           ▼
    TC1 ───●                 ●           │           │      Expected Result
    TC2 ───●                 │           │           │      Invalid Input
    TC3 ──────────────────── ●           │           │      Invalid Input
    TC4 ──────────────────────────────── ●           │      Fail
    TC5 ──────────────────────────────────────────── ●      Pass
    TC6 ────────────────────────────────────────────────●   Fail
```

**4) 분류 트리 기반 테스트 케이스**

| 파라미터 결정 (입력 조건) | 내용 |
|:----------------------|:-----|
| P1: 자료형(Type) = {정수, 비정수} | |
| P2: 정수 범위(Range) = {<10, 10-100, >100} | |
| 단, 비정수인 경우 P2는 적용되지 않음(N/A) | |

| 파라미터별 분류 트리 구성 | |
|:---------------------|:-----|
| 루트: 자료형(Type) | |
| 정수 → 하위 분기: 정수 범위(Range) = (<10 / 10-100 / >100) | |
| 비정수 → Range 미적용 (단말 노드) | |

| ID 부여 | |
|:-------|:-----|
| P1.v1 = 정수, P1.v2 = 비정수 | |
| P2.v1 = {10, P2.v2 = 10-100, P2.v3 = >100} | |

| TC ID | P2: 정수 범위 | 입력값(예) | 기대 결과 |
|:-----:|:------------|:---------|:---------|
| TC1 | (<10)(P2.v1) | 9 | Fail |
| TC2 | 10~100(P2.v2) | 10 | Pass |
| TC3 | | 100 | Pass |
| TC4 | >100(P2.v3) | 101 | Fail |
| TC5 | - (N/A) | "12.5" | Invalid Input |

**5) 테스트 수행**

- TC를 실제값으로 실행 후, 결과를 검증하여 결함을 분석
- 유효(Valid): (정수, 10~100) → TC2, TC3
- 비유효(Invalid): (정수, <10) → TC1 / (정수, >100) → TC4 / (비정수, -) → TC5

> TC를 대상 시스템의 입력조건을 분류, 트리구조로 표현하여 테스트 케이스 생성

---

### ⭐ 차별점 키워드

| 키워드 | 설명 |
|:------|:-----|
| **SW 실행 여부** | 동적 테스트는 실제 프로그램 실행, 정적 테스트는 미실행 |
| **명세 vs 구조** | 명세 기반(블랙박스)은 외부 관점, 구조 기반(화이트박스)은 내부 관점 |
| **동등 분할** | 입력 영역을 동등 클래스로 분할하여 대표값으로 테스트 |
| **분류 트리** | 입력조건을 트리 구조로 시각화하여 조합 테스트 케이스 도출 |

### 📈 상위 토픽 계층도

```
SW 테스트
├── 정적 테스트 (SW 미실행)
│   ├── Review
│   ├── Walk Through
│   └── Inspection
└── 동적 테스트 (SW 실행) ◀── 현재 위치
    ├── 명세 기반 테스트 (블랙박스)
    │   ├── 동등 분할
    │   ├── 분류 트리
    │   ├── 경계값 분석
    │   ├── 의사 결정 테이블
    │   ├── 상태 전이
    │   └── 유즈케이스
    └── 구조 기반 테스트 (화이트박스)
        ├── 구문 커버리지
        ├── 분기 커버리지
        ├── 조건 커버리지
        └── 경로 커버리지
```
