## 🎯 기술사 수준 설명

{: .highlight }
> **SW 대가산정 가이드 2025**: 수명주기 전 단계 적용, AI 서비스 도입 산정 체계 → 적정 대가 산정 실무 지침
> - (1) `적기절일 산품제투` (가이드 목적)
> - (2) `단커시` (AI 사업유형)
> - (3) `사서커구AI` (사업비 산정 절차)
> - ⭐ **차별점**: 2025년 개정 - AI 서비스 도입 유형별 산정 체계 신설

### 1. SW 제값받기, SW사업 대가산정의 개념 및 주요 개정사항

{: .highlight }
> **개념**: SW 수명주기 전 단계, 예산수립·발주·계약, 적정 대가 산정 기준
> - (개념) `기획·구현·운영 전 단계 대가산정 실무 지침`
> - (개정) `산사정` (산정절차개선, 사업추가, 정의추가)

#### 가. SW사업 대가산정 가이드 개념

| 구분 | 내용 |
|:-----|:-----|
| **정의** | 소프트웨어의 기획, 구현, 운영 전 수명주기에서 예산수립·발주·계약 시 적정 대가를 산정하기 위한 기준을 체계적으로 제시하는 실무 지침 |
| **적용 대상** | 국가·지자체 등 공공부문 SW사업 |
| **참고문헌** | 2025년 소프트웨어 사업 대가산정 가이드 (2025.08.11) |

#### 나. SW사업 대가산정 주요 개정사항 (2025년)

| 구분 | 주요개정 | 설명 |
|:-----|:--------|:-----|
| **산**정 절차 개선 | 인공지능 도입사업 대가 산정 체계 개선 | '전문작업비' 명칭을 '커스터마이징 작업비용'으로 변경하고, 유형별 작업 항목을 구체적으로 명시함 |
| **사**업 추가 | SW개발 운영 통합 사업 가이드 추가 | 클라우드 네이티브 기반의 SW사업 추진시 모듈별 구조 설계, 단계적 개통을 하는 사업과 시스템 개통 초기 단계의 신속한 장애 대응 및 안정화 지원을 위해 SW 개발·운영업무를 통합하여 추진하는 사업에 대한 가이드 추가 |
| **정**의 추가 | 일상운영 업무 내 "통합관리" 업무 정의 추가 | 연동·성능·버전 관리 활동을 '통합관리'로 묶어 명확히 정의 |

### 2. 소프트웨어 대가산정 가이드 목적

{: .highlight }
> **가이드 목적**: 객관적 기준 제공 + 공정·투명 집행
> - (객관적) `적기절일` (적용범위, 기준제공, 절차친화, 일관성)
> - (공정) `산품제투` (산정원칙, 품질, 제값지급, 투명성)

#### 가. 객관적 기준 제공 측면 가이드 목적 `적기절일`

| 구분 | 목적 | 설명 |
|:-----|:-----|:-----|
| **적**용 범위 | 수명주기 전 단계 적용 | 국가·지자체 등 공공부문이 기획-구현-운영 전 단계에서 활용하도록 범위를 명확히 함 |
| **기**준 제공 | 예산·발주·계약의 적정 대가 기준 | 사업의 예산수립·발주·계약 시 객관적 산정 기준을 제시함 |
| **절**차 친화 | 사용자 관점 절차 중심 설명 | 종전의 조문 해설식이 아닌 사용자 친화적·절차 중심으로 재구성해 현장 적용성을 높임 |
| **일**관성 | 기관·사업 간 산정 일관성 확보 | 통일된 기준을 통해 산정의 일관성·예측가능성을 높이고 이해관계자 간 혼선을 줄임 |

#### 나. 공정·투명 집행 측면 가이드 목적 `산품제투`

| 구분 | 목적 | 설명 |
|:-----|:-----|:-----|
| **산**정 원칙 | 합리·객관 산정 유도 | 왜곡을 줄이고 공정한 산정을 유도하여 집행의 신뢰성 확보 |
| **품**질 | 결과물 품질 향상 | 예산수립·발주·계약 시 활용할 객관적 산정 근거 제공 |
| **제**값지급 | 공정 보상·건전한 시장 조성 | 조문 나열이 아닌 절차 중심 안내로 현장 적용성 강화 |
| **투**명성·책무성 | 예산 집행의 투명성 강화 | 통일된 기준으로 예측가능성·비교가능성 향상, 혼선 최소화 |

---

### 3. 인공지능(AI) 서비스 도입 사업유형과 사업비 산정 절차

{: .highlight }
> **AI 서비스 도입**: 단순/커스터마이징/시스템통합 유형, 5단계 산정 절차
> - (유형) `단커시` (단순도입형, 커스터마이징형, 시스템통합형)
> - (커스터마이징) `기데모` (기본, 데이터구축, 모델)
> - (절차) `사서커구AI` (사전준비, 서비스이용료, 커스터마이징, 구축개발, AI산정)

#### 가. AI 서비스 도입 사업유형 `단커시`

**1) 사업유형과 비용 및 작업요소 간 관계**

```
┌─────────────────────────────────────────────────────────────────────┐
│            〈AI 서비스 도입 사업유형과 비용 및 작업요소 간 관계〉        │
├─────────────────────────────────────────────────────────────────────┤
│                  │  단순   │      커스터마이징형       │ 시스템  │
│   비용항목        │ 도입형  │  기본  │ 데이터 │ 모델  │ 통합형  │
├─────────────────────────────────────────────────────────────────────┤
│ 서비스 이용료     │   ○    │   ○   │   ○   │  ○   │   △   │
├─────────────────────────────────────────────────────────────────────┤
│ 커스터마이징     │        │   ○   │   ○   │  ○   │       │
│ 작업비용         │        │       │       │      │   △   │
│  - 요구사항분석/설계│       │   ○   │   ○   │  ○   │       │
│  - 데이터 구축    │        │       │   ○   │  ○   │       │
│  - 모델 구현 및 학습│       │       │       │  ○   │       │
│  - 검증 및 안정화  │        │   ○   │   ○   │  ○   │       │
├─────────────────────────────────────────────────────────────────────┤
│ 구축개발비용      │        │       │       │      │   ○   │
└─────────────────────────────────────────────────────────────────────┘
(○: 필수, △: 선택적)
```

**2) 유형별 상세 설명**

| 유형 | 설명 |
|:-----|:-----|
| **단**순 AI 서비스 도입형 | 공공 도입이 가능한 AI 서비스를 개발 또는 변형 없이 정기 이용료(구독료)를 지불하고 사용하는 사업유형 |
| **커**스터마이징형 - 기본 | 최소한의 커스터마이징 작업이 요구되는 유형. 기초 수준의 요구사항 분석 및 설계, 샘플 데이터셋 활용, 사전 학습된 모델 적용, 기초 수준의 검증 및 안정화 등의 작업 수행. ※ 해당 커스터마이징 작업이 서비스 이용료에 포함된 경우, 중복 지급 |
| **커**스터마이징형 - AI 데이터 구축 | 데이터 신규 구축 또는 재구축 등 데이터 구축 커스터마이징 작업이 요구되는 유형. 데이터 관련 요구사항 분석 및 설계, 수집/정제/가공/검수 등 데이터 구축 또는 재구축, 데이터 품질 검증 및 안정화 등의 작업 수행 |
| **커**스터마이징형 - AI 모델 | 기존 AI 모델 최적화, 새로운 AI 모델이나 알고리즘 개발 등의 커스터마이징 작업이 요구되는 유형. 모델 또는 알고리즘 관련 요구사항 분석 및 설계, 데이터 구축 또는 재구축, 모델 최적화 또는 파인튜닝, 모델 또는 알고리즘 개발, 검증 및 안정화 등의 작업 수행 |
| **시**스템 통합형 | 공공 도입이 가능한 AI 서비스를 도입하면서 발주기관의 도입 목적에 따라 커스터마이징 작업과 추가적인 소프트웨어 개발 및 시스템통합 작업이 병행되는 사업유형 |

**3) 서비스 도입 사업비 공식**

```
┌───────────────────────────────────────────────────────────────────┐
│  서비스 도입 사업비 = 서비스 총이용료 + 커스터마이징 작업비용        │
│                      + 구축·개발비용                              │
└───────────────────────────────────────────────────────────────────┘
```

#### 나. AI 서비스 도입 사업비 산정 절차 `사서커구AI`

```
┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐
│1. 사전준비│ → │2. 서비스 │ → │3. 커스터 │ → │4. 구축   │ → │5. AI서비스│
│          │    │  이용료  │    │ 마이징   │    │ 개발비용 │    │도입 사업비│
│          │    │  계산    │    │작업비용  │    │  계산    │    │  산정    │
└──────────┘    └──────────┘    └──────────┘    └──────────┘    └──────────┘
```

| 절차 | 추가활동 | 산출물 |
|:-----|:--------|:------|
| **1. 사**전준비 | - 도입 대상 서비스 식별 <br>- 세부 도입 서비스 항목과 서비스 도입 유형 결정 <br>- 산출물 서비스 도입 사업비 대상 서비스 및 추가 활동 항목 | 대상 서비스 및 추가 활동 항목 |
| **2. 서**비스 이용료 계산 | - 서비스 특성을 고려하여 사용기간 결정 <br>- 해당 서비스 가격표 또는 견적서 참고하여 이용료 계산 | 서비스 이용료 |
| **3. 커**스터마이징 작업비용 계산 | - 서비스 도입 시 필요한 커스터마이징 작업비용 항목 식별 <br>- 해당 서비스 가격표 또는 견적서, 유사 서비스 가격표 또는 견적서를 참고하여 커스터마이징 작업비용 계산 | 커스터마이징 작업비용 |
| **4. 구**축·개발 비용 계산 | - 서비스 도입 시 필요한 소프트웨어 개발 및 시스템통합 작업비용항목 식별 <br>- 기존 소프트웨어 개발비 산정 방식에 따라 기능점수 또는 투입공수 방식으로 구축·개발 비용 계산 | 구축·개발 비용 |
| **5. AI** 서비스 도입 사업비 산정 | - 서비스 도입 사업비 산정 <br>- 서비스 이용료 + 커스터마이징 작업비용 + 구축·개발비용 | 서비스 도입 사업비 |
