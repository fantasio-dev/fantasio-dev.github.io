### 📝 개념 정의

> **형상관리**: 소프트웨어 산출물의 변경을 체계적으로 관리하는 활동

| 구분 | 정의 (22자 이내) |
|:-----|:----------------|
| **형상관리** | 산출물 변경의 체계적 관리 |
| **기준선** | 공식 승인된 특정 시점 버전 |

### 쉬운 비유

| 게임 세이브 | 형상관리 기준선 |
|:-----|:-----|
| 중요한 시점에 세이브해두면 언제든 돌아갈 수 있음 | 개발 주요 시점마다 기준선 설정하면 해당 시점으로 복귀 가능 |

### 🏗️ 기술요소

#### 그룹 1: 형상관리 절차 `식통감기`

| 절차 | 설명 | 예시 |
|:-----|:-----|:-----|
| **식**별 | 관리 대상 항목 선정 및 고유번호 부여 | 소스코드, 설계서 |
| **통**제 | 변경 요청 검토 및 승인/거부 | CCB 회의 |
| **감**사 | 기준선 준수 및 무결성 검증 | 정기 감사 |
| **기**록 | 변경 이력 및 상태 보고 | 형상상태보고서 |

#### 그룹 2: 주요 기준선 `기분설제`

| 기준선 | 설명 | 예시 |
|:-----|:-----|:-----|
| **기**능적 기준선 | 요구사항 정의 완료 시점 | 개발계획서, WBS |
| **분**배적 기준선 | 요구사항 분할 완료 시점 | 요구사항정의서 |
| **설**계 기준선 | 설계 사양 완성 시점 | ERD, DFD, 클래스다이어그램 |
| **제**품 기준선 | 시험 완료 시점 | 시험보고서 |

## 🎯 기술사 수준 설명

{: .highlight }
> **형상관리**: 변경 추적 필요 → 형상식별, 형상통제, 형상감사, 형상기록 → 산출물 체계적 변경 관리 활동
> - (절차) `식통감기`
> - (기준선) `기분설제운`
> - ⭐ **차별점**: CCB(형상통제위원회)가 변경 승인/거부의 최종 권한 보유

---

### 📝 개념 정의

#### 🔑 핵심 키워드 3개

| 키워드 | 설명 |
|:------|:-----|
| **형상 항목(CI)** | Configuration Item, 관리 대상 산출물 |
| **형상 통제** | 변경 요청에 대한 승인/거부 결정 |
| **기준선(Baseline)** | 공식 승인된 특정 시점의 형상 항목 집합 |

---

#### 📖 등장배경

```
🌐 과거: 소프트웨어 산출물이 무분별하게 변경됨

💡 문제: "변경 이력 추적 없이 품질과 일관성을 유지할 수 있을까?"

✅ 해결: 형상관리 도입으로 체계적 변경 관리!
   → 형상 항목 식별 및 고유번호 부여
   → CCB를 통한 변경 승인 프로세스
   → 기준선 설정으로 복귀 지점 확보
```

---

#### 📝 형상관리의 정의

> **형상관리(SCM)**: 형상 항목을 식별하여 기능적·물리적 특성을 문서화하고, 변경을 제어하고, 변경 처리 상태를 기록·보고하고, 요구사항 부합 여부를 검증하는 기술적·관리적 활동

| 구분 | 정의 |
|:-----|:-----|
| **형상관리** | 소프트웨어 산출물 변경의 체계적 관리 활동 |
| **형상 항목** | 형상관리 대상이 되는 소프트웨어 산출물 단위 |
| **기준선** | 특정 시점에 공식 승인된 형상 항목의 집합 |

---

### 🏗️ 구성요소

#### 그룹 1: 형상관리 절차 `식통감기`

| 절차 | 핵심 활동 | 설명 |
|:-----|:-----|:-----|
| **식**별 (Identification) | 형상 항목 선정, 고유번호 부여 | 관리 대상 산출물을 식별하고 명명 규칙에 따라 고유 ID 부여 |
| **통**제 (Control) | 변경 요청 검토, 승인/거부 | CCB에서 변경 영향 분석 후 승인 여부 결정, 변경 이력 관리 |
| **감**사 (Audit) | 기준선 준수 검증, 무결성 확인 | 형상 항목이 기준선과 일치하는지, 변경 절차 준수 여부 검증 |
| **기**록 (Status Accounting) | 상태 기록, 보고서 작성 | 형상 항목의 현재 상태와 변경 이력을 기록하고 관계자에게 보고 |

#### 그룹 2: 형상관리 기준선(Baseline) `기분설제`

| 기준선 | 설정 시점 | 주요 산출물 |
|:-----|:-----|:-----|
| **기**능적 기준선 (Functional) | 요구사항 정의 완료 시점 | 개발계획서, 형상관리계획서, WBS |
| **분**배적 기준선 (Allocated) | 요구사항 서브시스템 분할 시점 | 요구사항정의서, SRS |
| **설**계 기준선 (Design) | 설계 사양 완성 시점 | ERD, DFD, Class Diagram |
| **제**품 기준선 (Product) | 통합/기능/성능 시험 완료 시점 | 시험보고서, 실행코드 |

---

### 📊 형상관리 기준선 전체 구조

| 단계 | 기준선 | 시점 | 산출물 |
|:-----|:-----|:-----|:-----|
| 계획 | 기능적 기준선 | 요구사항 정의 | 개발계획서, 형상관리계획서, WBS |
| 요구분석 | 분배적 기준선 | 서브시스템 분할 | 요구사항정의서 |
| 설계 | 설계 기준선 | 설계사양 완성 | ERD, DFD, Class Diagram |
| 구현 | 시험 기준선 | 시험 준비 수립 | 원시코드, 목적코드, 실행코드 |
| 통합 시험 | 제품 기준선 | 시험 완료 | 시험보고서 |
| 운영 | 운용 기준선 | 운영 이관 | 사용자/운영자 매뉴얼 |

---

### 🔧 형상관리 도구

| 구분 | 도구 | 설명 |
|:-----|:-----|:-----|
| **저장관리** | Git, SVN, CVS | 소스코드 변경 추적 및 버전 관리 |
| **원격저장소** | GitHub, GitLab, Bitbucket | 웹 기반 협업 및 코드 리뷰 |
| **변경관리** | CCB, 변경요청서, 형상상태보고서 | 변경 승인 절차 및 이력 문서화 |

---

### ⭐ 차별점 키워드

| 차별점 | 설명 |
|:------|:-----|
| **CCB (Configuration Control Board)** | 형상통제위원회, 변경 승인/거부의 최종 결정 권한 보유 |
| **기준선 = 복귀 지점** | 문제 발생 시 해당 기준선으로 롤백 가능 |
| **전 생명주기 적용** | 계획~운영까지 모든 단계에서 기준선 관리 |
