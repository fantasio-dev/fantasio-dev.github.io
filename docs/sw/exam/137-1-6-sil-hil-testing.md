### 🔑 핵심 키워드 3개

| 키워드 | 설명 |
|:------|:-----|
| **SIL** | 실제 하드웨어 없이 소프트웨어만으로 시뮬레이션 테스트 |
| **HIL** | 실제 하드웨어(ECU)를 포함해서 시뮬레이터와 함께 테스트 |
| **V-Model** | 개발 단계별 검증을 위한 MIL→SIL→PIL→HIL 순서 |

---

### 📖 등장배경

```
🌐 과거: 자동차, 항공기 등 임베디드 시스템은 실제 하드웨어로만 테스트했음

💡 문제: "실제 하드웨어로 테스트하면 비용도 높고, 위험한 상황은 테스트할 수 없잖아?"

✅ 해결: SIL과 HIL 테스팅으로 해결!
   → SIL: 소프트웨어만으로 먼저 검증 (비용 절감)
   → HIL: 실제 하드웨어와 시뮬레이터로 안전하게 검증
   → 위험한 시나리오도 가상으로 테스트 가능
```

---

### 📝 개념 정의

> **SIL**: 실제 하드웨어 없이 PC에서 소프트웨어만으로 시뮬레이션하는 테스트
> **HIL**: 실제 하드웨어(ECU)를 시뮬레이터에 연결해서 테스트

### 쉬운 비유

| SIL | HIL |
|:------|:------|
| 🎮 운전 게임으로 먼저 연습하는 것 | 🚗 운전 시뮬레이터에 실제 핸들/페달 연결해서 연습 |
| 컴퓨터 화면에서만 비행 시뮬레이션 | 실제 조종간을 시뮬레이터에 연결해서 테스트 |

---

### 🏗️ 기술요소

#### 그룹 1: SIL 테스트 절차 `모시통결자`

| 항목 | 설명 | 예시 |
|:-----|:-----|:-----|
| **모**델 준비 | Simulink, MATLAB 등에서 제어 알고리즘 작성 | 제어 로직 모델링 |
| **시**뮬레이션 환경 | Plant 모델을 소프트웨어로 구축 | 시스템 모델 시뮬레이션 |
| **통**합 테스트 | 제어기 코드와 Plant 모델 연결 후 시나리오 테스트 | 센서 신호, 환경 조건 주입 |
| **결**과 분석 | 성능 지표 측정 및 기대값 비교 | overshoot, settling time |

#### 그룹 2: HIL 테스트 절차 `하플연결`

| 항목 | 설명 | 예시 |
|:-----|:-----|:-----|
| **하**드웨어 준비 | 테스트 대상 제어기(ECU) 실제 구성 | 임베디드 보드, ECU |
| **플**랜트 모델 | 물리 시스템을 실시간 시뮬레이션 모델로 작성 | 엔진, 배터리, 모터 모델 |
| **연**결 구성 | 제어기 입력 신호를 시뮬레이터에서 생성 | 센서값 시뮬레이션 |
| **결**과 분석 | 안전 요구사항 만족 여부 및 성능 지표 분석 | ISO 26262 검증 |

---

### ⭐ 차별점 키워드

**SIL vs HIL**: SIL은 **소프트웨어만** 검증, HIL은 **실제 하드웨어 포함** 검증

---

### 📋 6하원칙 요약

| 구분 | SIL | HIL |
|:-----|:------|:------|
| **What** | SW만으로 폐루프 시뮬레이션 | 실제 HW와 시뮬레이터 통합 테스트 |
| **Why** | 초기 개발 단계 검증, 비용 절감 | 실제 환경에 가까운 검증 |
| **Who** | SW 개발자, 알고리즘 엔지니어 | HW/SW 통합 엔지니어 |
| **When** | 개발 초기, 알고리즘 검증 시 | 양산 전 최종 검증 시 |
| **Where** | 개발 PC, 시뮬레이션 환경 | HIL 테스트 장비, 시험실 |
| **How** | MATLAB, Simulink, C/C++ | dSPACE, NI, Vector 장비 |

## 🎯 기술사 수준 설명

{: .highlight }
> **SIL/HIL 테스팅**: SDV 개발 검증 필요 → V-Model, 폐루프, 실시간 → 임베디드 시스템 검증 기법
> - (SIL절차) `모시통결자` (모델준비, 시뮬레이션, 통합, 결과분석, 자동화)
> - (HIL절차) `하플연결` (하드웨어, 플랜트모델, 연결구성, 결과분석)
> - ⭐ **차별점**: SIL은 SW Only 검증, HIL은 실제 HW 포함 검증 (V-Model 단계별 적용)

---

### 📝 개념 정의

| 구분 | 정의 |
|:-----|:-----|
| **SIL** | 실제 하드웨어 없이 컨트롤러 SW와 Plant 모델을 PC에서 폐루프 시뮬레이션하는 테스트 |
| **HIL** | 실제 하드웨어(ECU)와 Plant 시뮬레이터를 연결하여 전체 시스템을 검증하는 테스트 |
| **SDV** | Software Defined Vehicle, 소프트웨어 중심 차량 개발 |

### 🏗️ 구성요소

#### 그룹 1: SIL 테스트 절차 `모시통결자`

| 항목 | 설명 | 예시 |
|:-----|:-----|:-----|
| **모**델/코드 준비 | Simulink, MATLAB, LabVIEW, C/C++에서 제어 알고리즘 모델 작성 | 제어기 로직 모델링 |
| **시**뮬레이션 환경 구축 | 제어 대상(Plant)을 소프트웨어로 시뮬레이션 | 가상 ECU 환경 구축 |
| **통**합 테스트 | PC에서 제어기 코드와 Plant 모델 연결, 다양한 시나리오 입력 주입 | 센서 신호, 환경 조건 테스트 |
| **결**과 분석 | 기대값과 시뮬레이션 결과 비교, 성능 지표 측정 | overshoot, settling time, steady-state error |
| **자**동화 및 반복 | 단위 테스트 → 통합 테스트 순으로 자동화, CI/CD 파이프라인 포함 | Jenkins, GitLab CI 연동 |

#### 그룹 2: HIL 테스트 절차 `하플연결`

| 항목 | 설명 | 예시 |
|:-----|:-----|:-----|
| **하**드웨어 준비 | 테스트 대상 제어기(ECU, 임베디드 보드)를 실제와 동일하게 구성 | 자동차 ECU, 항공 제어기 |
| **플**랜트 모델 준비 | 물리 시스템(엔진, 배터리, 모터)을 실시간 시뮬레이션 모델로 작성 | 엔진 Dynamics 모델 |
| **연**결 구성 | 제어기의 입력 신호(센서값)를 시뮬레이터에서 생성, 고장 시나리오 주입 | 잡음, 전압 강하 시뮬레이션 |
| **결**과 분석 | 기대값과 실제 HW 동작 비교, 안전 요구사항 만족 여부 평가 | ISO 26262, DO-178C 검증 |

### 📊 SIL vs HIL 비교표

| 비교 항목 | SIL (Software-in-the-Loop) | HIL (Hardware-in-the-Loop) |
|:---------|:---------------------------|:---------------------------|
| **테스트 대상** | 소프트웨어(제어 알고리즘) | 실제 하드웨어(ECU, 제어기) |
| **하드웨어 필요** | 불필요 (PC만으로 수행) | 필수 (실제 ECU 연결) |
| **실시간성** | 비실시간 또는 가상 시간 | 실시간 (RT deadline 준수) |
| **테스트 환경** | 개발 PC, 시뮬레이션 SW | HIL 테스트 장비 (dSPACE, NI) |
| **비용** | 저비용 | 고비용 (장비 필요) |
| **검증 수준** | 알고리즘 로직 검증 | 시스템 통합 검증 |
| **적용 시점** | 개발 초기 단계 | 양산 전 최종 검증 |
| **장점** | 빠른 반복, 저비용 | 실제 환경에 가까운 검증 |
| **단점** | HW 특성 미반영 | 장비 비용, 설정 복잡 |
| **대표 도구** | MATLAB/Simulink, LabVIEW | dSPACE, NI, Vector, ETAS |
| **표준** | - | ISO 26262, DO-178C |
| **활용 분야** | 알고리즘 개발, 단위 테스트 | 자동차, 항공, 로봇 검증 |

---

### ⭐ 차별점 키워드

| 키워드 | 설명 |
|:------|:-----|
| **SW Only vs HW 포함** | SIL은 SW만 검증, HIL은 실제 HW와 SW 통합 검증 |
| **V-Model 단계** | MIL→SIL→PIL→HIL 순서로 점진적 검증 수준 향상 |
| **실시간성** | SIL은 비실시간, HIL은 실시간 제약 조건 검증 |
| **안전 표준** | HIL은 ISO 26262(자동차), DO-178C(항공) 등 안전 표준 검증에 필수 |

### 📈 상위 토픽 계층도

```
SW공학
├── SW 테스팅
│   ├── 테스트 레벨
│   │   ├── 단위 테스트
│   │   ├── 통합 테스트
│   │   └── 시스템 테스트
│   ├── 임베디드 시스템 테스트
│   │   ├── MIL (Model-in-the-Loop)
│   │   ├── SIL (Software-in-the-Loop) ◀── 현재 위치
│   │   ├── PIL (Processor-in-the-Loop)
│   │   └── HIL (Hardware-in-the-Loop) ◀── 현재 위치
│   └── V-Model
└── 자동차 소프트웨어
    ├── SDV (Software Defined Vehicle)
    ├── ISO 26262 (기능안전)
    └── AUTOSAR
```
