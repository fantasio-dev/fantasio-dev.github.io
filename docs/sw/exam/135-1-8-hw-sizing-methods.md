```

💡 문제: "너무 작으면 느리고, 너무 크면 돈 낭비인데 어떻게 적정 규모를 알 수 있을까?"

✅ 해결: TTA에서 하드웨어 규모산정 지침을 만들어 표준화!

   → 비슷한 시스템 참조하거나
   → 시뮬레이션으로 미리 테스트
```

### 📝 개념 정의

> **하드웨어 규모산정**: 정보시스템에 필요한 서버 사양을 객관적으로 산정하는 방법

| 구분 | 정의 |
|:-----|:-----|
| **규모산정** | 시스템 요구사항을 하드웨어 스펙으로 변환 |
| **TTA 지침** | 국가 표준 하드웨어 규모산정 방법론 |

### 쉬운 비유

| 수식계산법 | 참조법 | 시뮬레이션법 |
|:-----|:-----|:-----|
| 레시피 보고 재료량 계산 | 옆집이 4인가족에 밥솥 6인용이래서 나도! | 미리 샘플 요리해보고 결정 |

---

### 🏗️ 기술요소

#### 그룹 1: 규모산정 방법 `수참시`

| 방법 | 설명 | 예시 |
|:-----|:-----|:-----|
| **수**식계산법 | 공식으로 용량 계산 후 보정치 적용 | 사용자 수 × 트랜잭션 = CPU 필요량 |
| **참**조법 | 유사 시스템과 비교하여 산정 | "저 회사랑 비슷하니까 같은 사양으로" |
| **시**뮬레이션법 | 가상 부하로 테스트 후 산정 | 성능 테스트 도구로 부하 테스트 |

#### 그룹 2: 규모산정 대상 `CPU메디스`

| 대상 | 설명 | 예시 |
|:-----|:-----|:-----|
| **CPU** | 처리 능력 규모 산정 | tpmC, MIPS |
| **메**모리 | 시스템/응용 S/W 메모리 사용량 | OS + WAS + 여유분 |
| **디**스크 | DB, 아카이브, 백업 용량 | 데이터 + 로그 + 백업 |
| **스**토리지 | 외장 스토리지 규모 | SAN, NAS 용량 |

## 🎯 기술사 수준 설명

{: .highlight }
> **HW 규모산정**: 용량·성능 요구사항 → CPU, 메모리, 디스크, 스토리지 → 시스템 요구사항 변환 방법
> - (방법) `수참시` - 수식계산법, 참조법, 시뮬레이션법
> - (대상) `CPU메디스` - CPU, 메모리, 디스크, 스토리지
> - ⭐ **차별점**: TTA 국가 표준(TTAK.KO-10.0292/R3) - 공공사업 규모산정의 객관적 근거 제공

### 🔑 핵심 키워드 3개

| 키워드 | 설명 |
|:------|:-----|
| **수식계산법** | 사용자 수 등 요소를 토대로 용량 수치를 계산하고 보정치를 적용하는 방법 |
| **참조법** | 업무량에 따라 기존 데이터를 토대로 유사 규모를 비교하여 산정하는 방법 |
| **시뮬레이션법** | 대상 업무에 대한 작업부하를 모델링하고 시뮬레이션하여 산정하는 방법 |

### 📝 개념 정의

> **하드웨어 규모산정**: 기본적인 용량과 성능 요구사항이 제시되었을 때, 그것을 시스템 요구사항으로 변환하는 방법

| 구분 | 정의 |
|:-----|:-----|
| **규모산정** | 용량/성능 요구를 HW 스펙으로 변환하는 과정 |
| **TTA 지침** | 정보시스템 하드웨어 규모산정 표준 (TTAK.KO-10.0292/R3) |

---

### 🏗️ 구성요소

#### 그룹 1: 규모산정 방법 `수참시`

| 방법 | 개념 | 장점 | 단점 |
|:-----|:-----|:-----|:-----|
| **수**식계산법 | 사용자 수 등 요소로 용량 계산 후 보정치 적용 | 명확성(근거 제시 가능), 간편성 | 보정치 오류 시 결과 왜곡 |
| **참**조법 | 업무량(사용자 수, DB 크기)에 따라 유사 시스템 비교 산정 | 신뢰성(기존 시스템 기반), 편리성 | 수학적 근거 미약 |
| **시**뮬레이션법 | 대상 업무 작업부하를 모델링하고 시뮬레이션하여 산정 | 정확성(실제 부하 기반) | 시간/비용 비효율성 |

#### 그룹 2: 규모산정 대상 `CPU메디스`

| 대상 | 설명 | 산정 요소 |
|:-----|:-----|:-----|
| **CPU** | 업무 처리를 위한 CPU 전체 규모 계산 후 서버 기종 선정 | tpmC, MIPS, SPECint |
| **메**모리 | 서버별 시스템 S/W, 응용프로그램 등 메모리 사용량 산정 | OS + WAS + App + 여유분 |
| **디**스크 | OS, 시스템 S/W, DB 데이터, 아카이브, 백업 영역 산정 | Data + Index + Log + Backup |
| **스**토리지 | CPU 기준 서버 규모에 따라 필요한 스토리지 규모 산정 | SAN, NAS, DAS 용량 |
